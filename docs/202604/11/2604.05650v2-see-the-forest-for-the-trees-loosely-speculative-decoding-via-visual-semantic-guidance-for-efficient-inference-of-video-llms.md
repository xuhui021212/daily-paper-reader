## 摘要
视频大语言模型（Video-LLMs）在视频理解方面表现卓越，但在自回归生成过程中面临高推理延迟的问题。投机解码（Speculative Decoding, SD）通过采用“草稿-验证”范式来缓解这一问题，但现有方法受限于僵化的精确匹配规则，严重限制了加速潜力。为了弥补这一差距，我们提出了 LVSpec，这是首个专为 Video-LLMs 设计的无需训练的宽松投机解码框架。基于生成过程是由稀疏的视觉相关锚点（要求严格验证）和大量的视觉无关填充词（允许宽松验证）共同主导的这一洞察，LVSpec 采用了一种轻量级的视觉相关 Token 识别方案来准确识别前者。为了进一步最大化接受率，我们还增加了一种位置偏移容忍机制，能够有效挽救位置不匹配但语义等价的 Token。实验表明，LVSpec 实现了高保真度和高速度：它在保持目标模型 99.8% 以上性能的同时，将 Qwen2.5-VL-32B 的推理速度提升了 2.70 倍，将 LLaVA-OneVision-72B 提升了 2.94 倍。值得注意的是，与现有的针对 Video-LLMs 的最先进（SOTA）免训练投机解码方法相比，它将平均接受长度和加速比分别提高了 136% 和 35%。

## Abstract
Video Large Language Models (Video-LLMs) excel in video understanding but suffer from high inference latency during autoregressive generation. Speculative Decoding (SD) mitigates this by applying a draft-and-verify paradigm, yet existing methods are constrained by rigid exact-match rules, severely limiting the acceleration potential. To bridge this gap, we propose LVSpec, the first training-free loosely SD framework tailored for Video-LLMs. Grounded in the insight that generation is governed by sparse visual-relevant anchors (mandating strictness) amidst abundant visual-irrelevant fillers (permitting loose verification), LVSpec employs a lightweight visual-relevant token identification scheme to accurately pinpoint the former. To further maximize acceptance, we augment this with a position-shift tolerant mechanism that effectively salvages positionally mismatched but semantically equivalent tokens. Experiments demonstrate that LVSpec achieves high fidelity and speed: it preserves >99.8 of target performance while accelerating Qwen2.5-VL-32B by 2.70x and LLaVA-OneVision-72B by 2.94x. Notably, it boosts the mean accepted length and speedup ratio by 136% and 35% compared to SOTA training-free SD methods for Video-LLMs.

### 1. 核心问题与研究动机
*   **核心问题**：视频大语言模型（Video-LLMs）在推理时面临极高的延迟，主要源于自回归解码机制以及海量的视觉 Token 处理。
*   **研究动机**：现有的加速方案“投机解码”（Speculative Decoding, SD）通常遵循“精确匹配”原则，即草稿模型生成的 Token 必须与目标模型完全一致才能被接受。这种过于僵化的规则导致大量语义正确但字面不符的 Token 被拒绝，限制了加速比。
*   **核心洞察**：作者发现视频理解任务的生成内容具有“双重性”：一小部分是与视觉内容强相关的“锚点 Token”（如具体的颜色、物体名），它们决定了事实准确性；大部分是视觉无关的“填充 Token”（如助词、连词），它们对语义影响较小。

### 2. 方法论
LVSpec 是一个**无需训练**的宽松投机解码框架，包含三个关键技术：
*   **视觉相关 Token 识别（Visual-Relevant Token Identification）**：
    *   在解码每一步，计算文本隐藏状态与视觉隐藏状态之间的余弦相似度。
    *   通过取 Top-N 最大相似度的平均值作为“视觉相关性得分”，精准识别出哪些 Token 是承载视觉信息的关键锚点。
*   **视觉语义引导的宽松验证（Visual-Semantic Guided Loose Verification）**：
    *   设定阈值 $\lambda$，对视觉相关性得分高的 Token 执行“严格验证”（必须精确匹配）；对得分低的填充 Token 执行“宽松验证”（允许不匹配时也接受，以换取速度）。
*   **位置偏移容忍机制（Position-Shift Tolerant, PST）**：
    *   针对视频描述中常见的“词序交换”或“冗余虚词”导致的不匹配，如果目标模型生成的 Token 出现在草稿序列的邻近范围内，则视为匹配成功，进一步提升接受长度。

### 6. 主要结论与发现
*   **加速效果显著**：在 Qwen2.5-VL-32B 上实现 **2.70×** 加速，在 LLaVA-OneVision-72B 上实现 **2.94×** 加速。

*   **超参数依赖**：对于不同视觉信息密度的任务，可能需要微调宽松参数 $\lambda$ 以达到速度与质量的最佳平衡。

*   **位置偏移范围**：PST 机制目前依赖于固定的草稿长度 $K$ 作为搜索窗口，对于更大跨度的位置偏移可能失效。

## 摘要
多模态大语言模型（MLLMs）近期在跨模态理解与生成方面展现出了卓越的能力。然而，视觉 Token 序列的快速增长——尤其是在长视频和流式场景中——对其可扩展性和实际部署构成了重大挑战。因此，我们推出了 POINTS-Long，这是一种受人类视觉系统启发、具有动态视觉 Token 缩放功能的原生双模式 MLLM。该模型支持两种互补的感知模式：专注模式（focus mode）和待机模式（standby mode），使用户能够在推理过程中动态地权衡效率与准确性。在细粒度视觉任务中，专注模式保持了最佳性能；而在长时通用视觉理解中，待机模式仅使用 1/40 至 1/10 的视觉 Token 即可保留 97.7% 至 99.7% 的原始准确率。此外，POINTS-Long 通过动态可拆卸的 KV 缓存（KV-cache）设计，原生支持流式视觉理解，从而能够高效维护超长视觉记忆。我们的工作为未来 MLLM 的设计提供了新的见解，并为自适应且高效的长时视觉理解奠定了基础。

## Abstract
Multimodal Large Language Models (MLLMs) have recently demonstrated remarkable capabilities in cross-modal understanding and generation. However, the rapid growth of visual token sequences--especially in long-video and streaming scenarios--poses a major challenge to their scalability and real-world deployment. Thus, we introduce POINTS-Long, a native dual-mode MLLM featuring dynamic visual token scaling inspired by the human visual system. The model supports two complementary perception modes: focus mode and standby mode, enabling users to dynamically trade off efficiency and accuracy during inference. On fine-grained visual tasks, the focus mode retains the optimal performance, while on long-form general visual understanding, the standby mode retains 97.7-99.7% of the original accuracy using only 1/40-1/10th of the visual tokens. Moreover, POINTS-Long natively supports streaming visual understanding via a dynamically detachable KV-cache design, allowing efficient maintenance of ultra-long visual memory. Our work provides new insights into the design of future MLLMs and lays the foundation for adaptive and efficient long-form visual understanding.

*   **核心挑战**：现有的视觉 Token 压缩技术往往面临三个困境：压缩率不足以应对数千帧视频、在效率与细粒度推理之间存在不可调和的折中（要么是视频专家，要么是图像专家）、不兼容现代推理框架（如 vLLM、SGLang）。

### 2. 论文提出的方法论
*   **核心思想**：在保留原始 MLLM 路径（Focus Mode）的基础上，引入一组可学习的“待机 Token”（Standby Tokens），通过并行路径蒸馏全局视觉信息。
*   **关键技术细节**：
    *   **双路径 ViT 架构**：在 Vision Transformer 中为待机 Token 复制并行的 MLP 层，并使用**非对称注意力掩码**。原始视觉 Token 仅自注意，而待机 Token 可以观察全量 Token，确保原始路径不受干扰。
    *   **时序建模**：在 ViT 末尾引入因果时序注意力（Causal Temporal Attention），对相邻 8 帧的待机 Token 进行联合建模，捕捉时序冗余。
    *   **两阶段训练策略**：
        *   **阶段 1（视觉蒸馏）**：冻结 LLM 和原始 ViT，仅训练待机 Token 和新增的并行层，强制模型将视觉信息压缩进极少数 Token 中。
        *   **阶段 2（模式适配）**：解冻 LLM，采用“双前向（2-forward）”训练，同时计算待机路径和专注路径的损失并取平均，使 LLM 能够理解两种尺度的输入。
    *   **可拆卸 KV 缓存（Detachable KV-cache）**：在流式推理中，当局部窗口满时，丢弃庞大的原始 Token 缓存，仅保留并迁移紧凑的待机 Token 缓存至“记忆库”，实现超长时记忆。

### 6. 主要结论与发现
*   **极高压缩比**：待机模式仅需 **1/40 至 1/10** 的 Token（每帧仅 8-32 Token），即可保留原始模型 **97.7% - 99.7%** 的性能。
*   **专注模式无损**：通过双路径设计，专注模式的图像理解能力完全没有下降，甚至略有提升。
*   **长程扩展性**：待机模式在处理 256 帧以上视频时表现出持续的性能增长，而普通模型往往在 64 帧后因上下文限制或长程衰减而性能停滞。
*   **推理加速**：在 SGLang 框架下，待机模式实现了高达 **6.2 倍** 的生成吞吐量提升。

### 7. 优点
*   **架构优雅**：通过非对称掩码和并行 MLP 解决了“性能与效率不可兼得”的矛盾，实现了真正的原生双模式。
*   **部署友好**：完全兼容 Flash-Attn 和现有的推理加速框架，不需要复杂的算子重写。
*   **流式创新**：提出的可拆卸 KV 缓存为处理超长视频流（如 30 分钟以上）提供了切实可行的工程方案。

### 8. 不足与局限
*   **空间感知损失**：定性分析显示，待机模式在涉及**细粒度空间识别**（如数数、小物体识别）时仍存在性能下降，无法完全替代专注模式。
*   **静态模式切换**：目前推理模式的切换是预定义的（用户选择），尚未实现模型根据问题复杂度自动、动态地决定哪些帧需要“专注”、哪些帧需要“待机”。
*   **训练成本**：虽然推理高效，但两阶段的后训练过程仍需要较大的算力投入（2.5 万 GPU 小时）。

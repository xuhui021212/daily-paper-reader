## 摘要
尽管最近取得了进展，视觉语言编码器仍面临两个核心局限性：(1) 语言与密集视觉特征之间的弱对齐，这损害了开放词汇语义分割等任务的性能；(2) 细粒度视觉表示的 Token 数量过多，限制了其在长视频上的可扩展性。本研究解决了这两个局限性。我们提出了 T-REN（文本对齐区域编码器网络），这是一种高效的编码器，可将视觉数据映射到一组紧凑的、与文本对齐的区域级表示（或称为区域 Token）。T-REN 通过在冻结的视觉骨干网络之上添加一个轻量级网络来实现，该网络经过训练，可将每个语义区域内的补丁级（patch-level）表示池化为区域 Token，并将其与区域级文本标注进行对齐。与视觉语言骨干网络相比，该设计仅增加了 3.7% 的额外参数，却在显著增强密集跨模态理解能力的同时，将 Token 数量降低了几个数量级。具体而言，T-REN 在 ADE20K 开放词汇分割上提升了 +5.9 mIoU，在 COCO 物体级文本-图像检索上提升了 +18.4% 的召回率，在 Ego4D 视频物体定位上提升了 +15.6% 的召回率，在 VSPW 视频场景解析上提升了 +17.6% mIoU；与此同时，与基于补丁的视觉语言骨干网络相比，其图像 Token 数量减少了 24 倍以上，视频 Token 数量减少了 187 倍以上。代码和模型可在 https://github.com/savya08/T-REN 获取。

## Abstract
Despite recent progress, vision-language encoders struggle with two core limitations: (1) weak alignment between language and dense vision features, which hurts tasks like open-vocabulary semantic segmentation; and (2) high token counts for fine-grained visual representations, which limits scalability to long videos. This work addresses both limitations. We propose T-REN (Text-aligned Region Encoder Network), an efficient encoder that maps visual data to a compact set of text-aligned region-level representations (or region tokens). T-REN achieves this through a lightweight network added on top of a frozen vision backbone, trained to pool patch-level representations within each semantic region into region tokens and align them with region-level text annotations. With only 3.7% additional parameters compared to the vision-language backbone, this design yields substantially stronger dense cross-modal understanding while reducing the token count by orders of magnitude. Specifically, T-REN delivers +5.9 mIoU on ADE20K open-vocabulary segmentation, +18.4% recall on COCO object-level text-image retrieval, +15.6% recall on Ego4D video object localization, and +17.6% mIoU on VSPW video scene parsing, all while reducing token counts by more than 24x for images and 187x for videos compared to the patch-based vision-language backbone. The code and model are available at https://github.com/savya08/T-REN.

现代视觉语言（VL）编码器（如 CLIP、DINOv3）面临两个核心瓶颈：
*   **密集对齐能力弱：** 语言与密集视觉特征（如像素或补丁级特征）之间的对齐不够精确，导致在开放词汇语义分割和物体定位任务中表现不佳。
*   **可扩展性受限：** 基于补丁（patch）的表示会产生数以千计的 Token，在处理长视频或大规模图像库时，计算和内存开销巨大。
*   **研究动机：** 作者认为补丁不是密集视觉语言建模的理想单位（太细碎且数量多），因此提出将表示单位从“补丁”转向“语义区域”，通过学习紧凑且与文本对齐的**区域 Token（Region Tokens）**来同时解决对齐和效率问题。

#### 2. 方法论
T-REN 在冻结的视觉骨干网络（如 DINOv3）之上添加了一个轻量级模块：
*   **核心思想：** 利用点提示（Point Prompts）作为查询（Queries），通过交叉注意力机制将语义相关的补丁特征池化为区域 Token，并将其投影到文本嵌入空间。
*   **关键技术细节：**
    *   **多 Token 预测：** 每个点提示生成 $k=3$ 个区域 Token，以捕捉物体的层级结构（如“车轮”与“整车”），解决前作 REN 的部分-整体歧义。
    *   **区域池化与对齐：** 使用 2 层解码器结构，通过交叉注意力进行空间池化，并通过 MLP 将结果投影到文本空间。
    *   **Token 合并策略：** 
        *   **帧内合并：** 合并余弦相似度高或掩码 IoU 重叠大的 Token，减少冗余。
        *   **帧间合并（视频）：** 将连续帧中的区域 Token 聚合为“轨迹 Token（Track Tokens）”，实现极高的视频压缩率。
*   **训练目标：** 采用四种损失函数联合训练：
    1.  **视觉对比损失：** 鼓励同一区域的 Token 聚类。
    2.  **文本对比损失：** 实现区域 Token 与类别文本的对齐。
    3.  **蒸馏损失：** 确保区域 Token 不偏离预训练骨干网络的特征空间。
    4.  **掩码监督损失：** 加速交叉注意力掩码的收敛。

*   **训练细节：** 在 5 个分割数据集（COCOStuff, SA-1B 等）的混合数据上训练，迭代 60,000 次，Batch Size 为 16。
*   **效率：** 在 512p 分辨率下，T-REN 的推理延迟仅比 DINOv3 增加约 1.7ms（使用 16x16 提示网格时）。

*   **实验规模：** 涵盖了从静态图像到长视频、从密集分割到高级推理的多种任务，实验设计非常全面。
*   **消融实验：** 极其充分。作者分别验证了区域池化、文本对齐、Token 合并策略、多 Token 预测（k 值）以及输入分辨率对性能的影响。
*   **客观性：** 采用了零样本（Zero-shot）评估协议，并专门针对训练集中未见过的类别进行了泛化性测试，证明了结果的客观性和鲁棒性。

#### 6. 主要结论与发现
*   **性能飞跃：** 在 ADE20K 上提升了 **+5.9 mIoU**，在视频场景解析上提升了 **+17.6% mIoU**。
*   **极致压缩：** 在保持高性能的同时，将图像 Token 数量减少了 **24.4 倍**，视频 Token 减少了 **187.5 倍**（Ego4D）甚至 **254.5 倍**（VSPW）。

#### 7. 优点
*   **高效且轻量：** 极小的参数增量换取了巨大的性能和效率提升。
*   **层级表达：** 通过每点多 Token 设计，成功捕捉了物体的部分与整体关系。
*   **存储友好：** 区域 Token 的数量不随图像分辨率增加而剧增，非常适合大规模索引和检索。

#### 8. 不足与局限
*   **骨干网络依赖：** T-REN 的上限受限于冻结的视觉骨干网络（如 DINOv3），若骨干网络本身存在偏差，T-REN 难以完全消除。

*   **端到端潜力：** 目前是作为插件使用，未来若能进行全参数端到端训练，可能会有更强的表现。

## 摘要
视频大语言模型（Video LLMs）在视频理解任务上表现出色，但由于视觉 Token 数量庞大，面临着高昂的推理成本。我们提出了 KiToke，这是一种无需训练且与查询无关的 Token 压缩方法，在保留关键视觉信息的同时减少了时空冗余。我们的方法利用基于核的冗余度量全局估计 Token 多样性，实现了在极端 Token 预算下依然有效的内容自适应选择，并进一步引入了轻量级的时序区间构建和区间感知 Token 合并，以保持时序连贯性。与以往依赖局部或片段级启发式方法不同，KiToke 显式地捕捉整个视频的全局冗余，从而实现了更高效的 Token 利用。在多个视频理解基准测试和 Video LLM 骨干网络上的广泛实验表明，KiToke 始终优于现有的无需训练的压缩方法，尤其是在保留率低至 1% 的激进情况下，性能提升尤为显著。

## Abstract
Video Large Language Models (Video LLMs) achieve strong performance on video understanding tasks but suffer from high inference costs due to the large number of visual tokens. We propose KiToke, a training-free, query-agnostic token compression approach that reduces spatiotemporal redundancy while preserving critical visual information. Our method estimates token diversity globally using a kernel-based redundancy measure, enabling content-adaptive selection that remains effective under extreme token budgets, and further introduces a lightweight temporal interval construction with interval-aware token merging to maintain temporal coherence. Unlike prior methods that rely on local or segment-level heuristics, KiToke explicitly captures global redundancy across an entire video, leading to more efficient token utilization. Extensive experiments on multiple video understanding benchmarks and Video LLM backbones demonstrate that KiToke consistently outperforms existing training-free compression methods, with particularly large gains at aggressive retention ratios down to 1%.

*   **研究动机**：视频大语言模型（Video LLMs）在处理长视频时会产生海量的视觉 Token，导致推理成本（计算量和显存占用）呈平方级增长。
*   **核心挑战**：现有的 Token 压缩方法在极低保留率（如 1%）下性能会剧烈下降，且往往依赖局部启发式规则，难以有效处理跨时空的全局冗余，或在压缩过程中破坏了视频的时序连贯性。
*   **整体含义**：本文提出了一种名为 **KiToke** 的无需训练（Training-free）、与查询无关（Query-agnostic）的压缩框架，旨在通过全局多样性估计和时序感知合并，在大幅降低 Token 数量的同时保留关键视觉信息。

### 2. 方法论
KiToke 主要包含三个核心技术模块：
*   **基于核的多样性估计（Kernel-based Diversity Estimation）**：
    *   利用**核密度估计（KDE）**的思想，通过高斯核函数计算每个 Token 在全局嵌入空间中的密度。
    *   密度越高代表冗余度越高，多样性得分定义为密度的倒数。这使得模型能从全局视角识别并抑制视频中重复的背景或冗余动作。
*   **时序区间构建（Temporal Interval Construction）**：
    *   计算帧间差异，结合了“对应位置差异”和“最佳匹配差异”（考虑运动位移）。
    *   通过检测帧间差异相对于局部动态的**突变（Deviation）**来自动划分时间区间，而非使用单一全局阈值，从而实现内容自适应的分割。
*   **区间感知 Token 合并（Interval-aware Token Merging）**：
    *   在每个自适应区间内，将未选中的 Token 合并到最相似的选中 Token 中。
    *   合并过程采用多样性得分进行加权平均，确保独特的视觉特征在聚合表示中占主导地位，同时避免跨越语义边界（区间边界）的错误合并。

### 6. 主要结论与发现
*   **性能领先**：KiToke 在所有基准测试和保留率下均优于现有方法，尤其在 **1% 极端保留率**下，性能提升最为显著（比次优方法高出约 5%）。
*   **高效性**：相比于基于聚类的复杂方法，KiToke 的计算开销极低，能显著缩短首 Token 延迟（Prefill Time）。
*   **冗余抑制**：实验发现，适当的压缩有时甚至能通过去除视觉噪声（如背景干扰）来提升模型在某些任务上的表现。

### 7. 优点
*   **全局冗余捕捉**：突破了局部窗口限制，能有效处理长视频中反复出现的场景。
*   **内容自适应**：时序区间构建能灵敏捕捉语义转折，保护了视频的时序结构。
*   **即插即用**：无需任何微调或架构修改，兼容性极强。
*   **极端鲁棒性**：在 Token 预算极度匮乏的情况下，依然能通过多样性采样保留关键证据。

### 8. 不足与局限
*   **查询无关性限制**：由于不参考用户问题（Query），在某些极小区域包含关键答案且该区域视觉不显著的情况下，可能会误删关键 Token。
*   **超参数依赖**：虽然实验证明参数不敏感，但仍需针对不同模型微调核带宽和区间阈值以达到最优效果。
*   **架构敏感度**：在某些高度依赖特定位置编码（如 Interleaved-MRoPE）的模型上，极端压缩带来的性能损失仍比普通架构略大。

## 摘要
多模态大语言模型（MLLMs）通过提高输入保真度实现了更强的视觉理解能力，但由此产生的视觉 token 增长使得同时维持高空间分辨率和长时序上下文变得成本极高。我们认为，瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量。为此，我们提出了 ResAdapt，这是一种输入端自适应框架，能够在编码前学习每一帧应分配的视觉预算。ResAdapt 将轻量级分配器（Allocator）与未经修改的 MLLM 主干网络相结合，使主干网络在接收算子转换后的输入时，仍能保留其原生的视觉 token 接口。我们将分配过程建模为上下文老虎机（contextual bandit），并使用成本感知策略优化（CAPO）来训练分配器，该方法将稀疏的 rollout 反馈转化为稳定的准确率-成本学习信号。在预算受限的视频问答、时序定位和图像推理任务中，ResAdapt 改善了低预算运行点的表现，且通常处于或接近效率-准确率前沿，在激进压缩下的推理密集型基准测试中提升最为显著。值得注意的是，在相同的视觉预算下，ResAdapt 支持的帧数增加了多达 16 倍，同时带来了超过 15% 的性能提升。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

### 1. 核心问题与整体含义（研究动机和背景）
多模态大语言模型（MLLM）在处理视频时面临**“视觉 Token 爆炸”**的挑战：提高空间分辨率能捕捉细节，增加帧数能理解长时序，但两者同时提升会导致计算开销不可接受。
*   **现有方案缺陷**：模型端压缩（如 Token 合并/剪枝）发生在编码后，丢失的细节无法恢复，且破坏了推理加速算子的兼容性；输出端代理推理（如迭代缩放）延迟高，且依赖初始的粗糙视图。
*   **研究动机**：作者认为效率瓶颈在于编码器接收的**原始像素总量**。因此提出在编码前进行“输入端自适应”，动态分配每帧的视觉预算。

### 2. 论文提出的方法论：ResAdapt 框架
ResAdapt 引入了一个轻量级的**分配器（Allocator）**，在不改变 MLLM 主干网络的情况下，实现单次前向传递的按需分配。
*   **核心思想**：将每帧的分辨率分配建模为一个**上下文老虎机（Contextual Bandit）**问题。分配器根据文本查询（Query）和视频的粗糙特征，预测每帧的缩放因子 $s_t$。
*   **关键技术细节**：
    *   **分配器架构**：基于 SmolVLM 的轻量级设计，利用时间自注意力（Temporal Self-attention）和查询交叉注意力（Cross-attention）捕捉时空冗余和查询相关性。
    *   **Beta 策略**：使用 Beta 分布参数化连续的缩放动作，支持在 $[s_{min}, s_{max}]$ 范围内灵活调整。
    *   **CAPO（成本感知策略优化）**：为了防止策略坍缩至最小预算，CAPO 引入了**动态成本基准（Dynamic Pivot）**和**非对称奖励整形**。对高效且正确的回答给予奖励，对高能耗且错误的回答给予重罚。
    *   **时序相似性正则化（$L_{sim}$）**：惩罚相邻相似帧同时获得高预算的行为，鼓励差异化分配。

### 5. 实验数量与充分性
*   **实验规模**：论文涵盖了从 32 帧到 128 帧的不同时序长度实验，并在 10 余个主流基准测试上进行了验证。
*   **消融实验**：非常充分。包括奖励设计（CAPO vs 直接成本惩罚）、分布类型（Beta vs 正态分布）、时序正则化（$L_{sim}$）的作用、以及算子泛化性（缩放策略迁移至丢帧策略）的测试。

### 6. 主要结论与发现
*   **效率前沿**：ResAdapt 在极低预算（约 10% Token 保留率）下表现优异，尤其在推理密集型任务（如 VideoMMMU）上提升显著。
*   **时空置换**：在相同 Token 预算下，ResAdapt 可处理 **16 倍**于基准模型的帧数，性能提升超过 15%。
*   **主动感知**：分配器学会了“主动感知”，能自动将预算集中在场景变换、文字出现或关键动作发生的帧上。
*   **算子泛化**：虽然是在缩放（Resize）算子上训练的，但该策略可无缝迁移至帧选择（Frame Selection）任务。

### 7. 优点
*   **硬件友好**：不改变主干网络结构，完美兼容 FlashAttention、vLLM 等现有的推理加速栈。
*   **查询感知**：相比于通用的视频压缩，ResAdapt 能根据问题寻找关键帧（如针对数学题寻找包含图表的帧）。
*   **单次推理**：避免了代理类方法（Agentic methods）多轮交互带来的高延迟。

### 8. 不足与局限
*   **固定开销**：分配器本身虽然轻量，但在处理极短视频（如 <32 帧）时，其前置计算开销可能会抵消部分 Token 节省带来的收益。
*   **开环限制**：分配决策是在编码前一次性完成的（开环），如果分配器初始判断错误（漏掉了关键细节），后续推理过程无法纠正。
*   **跨模态泛化偏差**：在视频上训练的策略在纯图像任务上的表现参差不齐，未能实现全场景的完美迁移。

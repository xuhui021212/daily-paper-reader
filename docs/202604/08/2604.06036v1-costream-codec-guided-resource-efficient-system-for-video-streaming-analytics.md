## 摘要
视频流分析是视觉语言模型服务中的关键工作负载，但多模态推理的高昂成本限制了其可扩展性。现有系统通过利用视频流中的时空冗余来降低推理成本，但它们往往仅针对视觉 Transformer (ViT) 或大语言模型 (LLM) 进行局部优化，未能充分挖掘端到端的优化潜力。此外，现有方法在识别冗余时会产生显著开销，无论是通过离线分析与训练，还是高昂的在线计算，这使得它们难以适应动态的实时视频流。我们提出了 CoStream，这是一个编解码器引导的流式视频分析系统。该系统基于一个关键观察：视频编解码器在压缩过程中已经作为副产品提取了每个视频流的时空结构。CoStream 将这些编解码器元数据视为低成本的运行时信号，从而统一了视频解码、视觉处理和 LLM 预填充阶段的优化；同时，直接在压缩比特流上操作也带来了传输量减少的固有优势。这驱动了 ViT 编码前的编解码器引导补丁剪枝以及 LLM 预填充期间的选择性键值缓存刷新，两者均为完全在线运行，无需离线训练。实验表明，与最先进的基准方法相比，CoStream 实现了高达 3 倍的吞吐量提升和高达 87% 的 GPU 计算量削减，同时保持了具有竞争力的准确率，F1 分数仅下降 0-8%。

## Abstract
Video streaming analytics is a crucial workload for vision-language model serving, but the high cost of multimodal inference limits scalability. Prior systems reduce inference cost by exploiting temporal and spatial redundancy in video streams, but they target either the vision transformer (ViT) or the LLM with a limited view, leaving end-to-end opportunities untapped. Moreover, existing methods incur significant overhead to identify redundancy, either through offline profiling and training or costly online computation, making them ill-suited for dynamic real-time streams.   We present CoStream, a codec-guided streaming video analytics system built on a key observation that video codecs already extract the temporal and spatial structure of each stream as a byproduct of compression. CoStream treats this codec metadata as a low-cost runtime signal to unify optimization across video decoding, visual processing, and LLM prefilling, with transmission reduction as an inherent benefit of operating directly on compressed bitstreams. This drives codec-guided patch pruning before ViT encoding and selective key-value cache refresh during LLM prefilling, both of which are fully online and do not require offline training. Experiments show that CoStream achieves up to 3x throughput improvement and up to 87% GPU compute reduction over state-of-the-art baselines, while maintaining competitive accuracy with only 0-8% F1 drop.

随着视觉语言模型（VLM）在监控、交通控制和工业自动化中的广泛应用，视频流分析面临着巨大的计算挑战。核心矛盾在于：

*   **计算冗余严重**：视频流在空间（相邻像素）和时间（相邻帧）上存在大量冗余，且滑动窗口推理导致重叠区域被反复计算。
*   **现有方法局限**：目前的优化技术要么仅针对 ViT 或 LLM 进行局部优化，要么依赖昂贵的离线训练或高开销的在线分析，难以应对动态变化的实时视频流。

#### 2. 方法论：核心思想与关键技术
CoStream 的核心思想是**利用视频编解码器（Codec）在压缩过程中产生的元数据（如运动矢量和残差）作为低成本的运行时信号**，来统一引导端到端的推理优化。

*   **硬件加速编解码处理**：利用 GPU 硬件解码器（如 NVIDIA NVDEC）进行单次解码并缓存，消除重叠窗口间的重复解码开销，并直接提取压缩域元数据。
*   **运动矢量引导的 Token 剪枝**：
    *   **原理**：通过分析 P 帧中的运动矢量（MV）和残差（Residual），识别视频中的静态或可预测区域。
    *   **操作**：将块级运动信号映射到 ViT 的补丁（Patch）网格上。若某区域运动量低于阈值，则在进入 ViT 编码前将其剪枝，从而减少视觉编码器的计算量和进入 LLM 的 Token 数量。
*   **选择性 KV 缓存（KVC）刷新**：
    *   **原理**：滑动窗口中的重叠帧虽然视觉内容相同，但其在 LLM 中的语义上下文会随窗口移动而漂移。
    *   **操作**：CoStream 利用帧类型（I 帧/P 帧）进行决策。将 I 帧作为“锚点”进行 KVC 刷新（重新计算），而对 P 帧的 Token 进行复用，并通过旋转位置嵌入（RoPE）进行位置修正，以保持语义一致性。

#### 6. 主要结论与发现
*   **性能提升**：CoStream 实现了高达 **3 倍** 的吞吐量提升，端到端延迟显著降低。

#### 7. 优点
*   **端到端协同**：打破了以往只优化 ViT 或 LLM 的孤岛模式，实现了从传输、解码到推理的全流程优化。
*   **极低开销**：巧妙利用编解码器现有的“副产品”信号，无需额外的离线训练或复杂的在线特征计算。
*   **部署友好**：基于生产级框架（vLLM）实现，且不依赖特定任务的语义，具有良好的通用性。

#### 8. 不足与局限
*   **硬件绑定**：目前高度依赖 NVIDIA 的硬件解码特性（NVDEC），在其他硬件平台（如国产芯片或移动端）上的移植性需进一步验证。

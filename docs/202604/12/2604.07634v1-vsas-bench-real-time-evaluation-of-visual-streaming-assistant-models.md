## 摘要
流式视觉语言模型（VLMs）在给定指令提示和在线输入帧流的情况下持续生成响应。这是实时视觉助手的核心机制。现有的 VLM 框架主要在离线设置下评估模型。相比之下，流式 VLM 的性能取决于纯视频理解之外的其他指标，包括反映模型响应及时性的主动性（proactiveness），以及捕捉其响应随时间推移的鲁棒性的一致性（consistency）。为了解决这一局限性，我们提出了 VSAS-Bench，这是一个针对视觉流式助手的新框架和基准测试。与以往主要对视频输入采用单轮问答的基准测试不同，VSAS-Bench 具有时间密集的标注，在不同输入领域和任务类型中包含超过 18,000 条标注。我们引入了标准化的同步和异步评估协议，以及能够隔离并衡量流式 VLM 不同能力的指标。利用该框架，我们对近期的视频和流式 VLM 进行了大规模评估，分析了在内存缓冲区长度、内存访问策略和输入分辨率等关键设计因素下的准确率-延迟权衡，并得出了一些实用的见解。最后，我们通过实验表明，传统的 VLM 可以在无需额外训练的情况下适配流式设置，并证明这些适配后的模型优于近期的流式 VLM。例如，在异步协议下，Qwen2-VL-7B 的表现比我们基准测试中最好的流式 VLM Dispider 高出 3%。基准测试和代码将在 https://github.com/apple/ml-vsas-bench 发布。

## Abstract
Streaming vision-language models (VLMs) continuously generate responses given an instruction prompt and an online stream of input frames. This is a core mechanism for real-time visual assistants. Existing VLM frameworks predominantly assess models in offline settings. In contrast, the performance of a streaming VLM depends on additional metrics beyond pure video understanding, including proactiveness, which reflects the timeliness of the model's responses, and consistency, which captures the robustness of its responses over time. To address this limitation, we propose VSAS-Bench, a new framework and benchmark for Visual Streaming Assistants. In contrast to prior benchmarks that primarily employ single-turn question answering on video inputs, VSAS-Bench features temporally dense annotations with over 18,000 annotations across diverse input domains and task types. We introduce standardized synchronous and asynchronous evaluation protocols, along with metrics that isolate and measure distinct capabilities of streaming VLMs. Using this framework, we conduct large-scale evaluations of recent video and streaming VLMs, analyzing the accuracy-latency trade-off under key design factors such as memory buffer length, memory access policy, and input resolution, yielding several practical insights. Finally, we show empirically that conventional VLMs can be adapted to streaming settings without additional training, and demonstrate that these adapted models outperform recent streaming VLMs. For example, Qwen3-VL-4B surpasses Dispider, the best streaming VLM on our benchmark, by 3% under the asynchronous protocol. The benchmark and code will be available at https://github.com/apple/ml-vsas-bench.

### VSAS-BENCH：视觉流式助手模型的实时评估

#### 1. 核心问题与整体含义
随着大语言模型（LLM）向多模态（VLM）演进，实时视觉助手（如助盲设备、烹饪助手）成为核心应用场景。然而，现有的 VLM 评估框架大多处于**离线设置**，即模型处理完整的视频剪辑而非连续的视频流。这种评估方式忽略了实时应用中的两个关键要素：
*   **主动性（Proactiveness）：** 模型能否在视觉流推进时及时做出响应。
*   **一致性（Consistency）：** 模型在连续帧之间的回答是否稳定且不自相矛盾。
**VSAS-BENCH** 的提出旨在填补这一空白，通过模拟真实的异步流式环境，衡量模型在准确率与延迟之间的权衡。

#### 2. 方法论
论文提出了一个全新的评估框架，包含以下核心组件：
*   **任务分类：**
    *   **Present（当前任务）：** 识别当前正在发生的事件。
    *   **Cumulative（累积任务）：** 对过去已发生的序列进行推理或总结。
    *   **Future（未来任务）：** 基于当前上下文预测接下来的步骤。
*   **评估协议：**
    *   **同步协议（Synchronous）：** 模型与相机流步调一致，不计推理延迟，主要衡量理解能力。
    *   **异步协议（Asynchronous）：** 模拟真实硬件环境。相机以固定帧率产生图像存入缓冲区，模型异步读取。若模型推理慢，缓冲区溢出则会导致丢帧，直接影响准确率。
*   **核心指标：**
    *   **平均准确率（A）：** 使用 GPT-5 作为裁判，对比模型输出与密集标注的语义一致性。
    *   **平均一致性（C）：** 基于编辑距离（LCS）衡量模型在连续时间步输出的稳定性。
*   **适配技术：** 提出一种**无需训练**的方法，通过“内存缓冲区”和“内存策略”（如滑动窗口 SW、均匀采样 U、混合策略 SW+U）将普通视频 VLM 转化为流式 VLM。

#### 3. 实验设计
*   **数据集：** 包含 92 个视频（48 个新录制，44 个源自 STAR、Perception-Test 等），总计超过 **18,000 条密集标注**（每秒 1 帧）。
*   **对比方法：**
    *   **闭源 API 模型：** GPT-5 (Low/Medium/High reasoning)。
    *   **开源视频 VLM：** InternVL3 系列、Qwen3-VL 系列（2B 到 38B）。
    *   **专用流式 VLM：** VideoLLM-Online、FlashVStream、Dispider、StreamBridge。
*   **消融实验：** 涵盖了内存缓冲区大小、输入图像分辨率、硬件加速器类型以及内存访问策略。

#### 4. 资源与算力
*   **硬件：** 统一使用 NVIDIA H100 GPU 进行评估，部分消融实验对比了 A100。
*   **配置：** 2B/4B 模型使用单卡，8B 模型使用双卡，32B/38B 模型使用四卡。
*   **时长：** 以 Qwen3-VL 为例，2B 模型完成全量评估需 8 GPU 小时，8B 模型需 12 GPU 小时。
*   **API 延迟：** 记录了 GPT-5 的网络 API 延迟（约 40-60 秒），并分析了其对异步评估性能的负面影响。

#### 5. 实验数量与充分性
论文进行了大规模且多维度的实验：
*   **规模：** 评估了 10 余种主流模型，涵盖了从轻量级（2B）到巨量级（32B+）的参数梯度。
*   **维度：** 不仅对比了模型性能，还深入探讨了硬件（A100 vs H100）对实时准确率的影响。
*   **客观性：** 标注经过专家人工审核（10% 纠错率），GPT-Judge 与人工标注的一致性（Cohen’s κ）高达 0.91。
*   **公平性：** 所有模型在相同的推理框架（CUDA 12.4, FlashAttention 2）下运行，确保了延迟测量的公正。

#### 6. 主要结论与发现
*   **小模型的逆袭：** 在异步协议下，**4B 规模的模型表现往往优于 32B 模型**。原因是小模型推理极快，能处理更多新鲜帧，而大模型因高延迟导致处理的是“过时”信息。
*   **适配优于专用：** 简单的“视频 VLM + 内存策略”适配方案，在性能上超过了专门针对流式任务微调的模型（如 Qwen3-VL-4B 超过了 Dispider）。
*   **权衡关系：** 增加内存缓冲区或图像分辨率虽能提升单次推理准确率，但会因增加延迟而在异步流式场景下降低整体表现。
*   **一致性趋势：** 专用流式模型在输出稳定性上通常优于通用视频模型，但准确率上限较低。

#### 7. 优点
*   **真实性：** 异步评估协议极大地贴近了移动端或机器人等边缘设备的真实部署环境。
*   **标注密集：** 1.8 万条标注远超同类基准（如 RTV-Bench），能精细捕捉模型在每一秒的反应。
*   **实用性：** 提出的“训练无关”适配方案为开发者提供了低成本将现有 VLM 转化为流式助手的路径。

#### 8. 不足与局限
*   **视频多样性：** 虽然标注密集，但 92 个视频的总量在覆盖极端长尾场景方面仍有提升空间。
*   **裁判偏差：** 依赖 GPT-5 作为裁判可能存在对特定风格回复的偏好，尽管论文通过人工验证缓解了此问题。
*   **任务范围：** 目前主要集中在视觉感知和简单推理，尚未深入涉及复杂的多轮交互对话逻辑。

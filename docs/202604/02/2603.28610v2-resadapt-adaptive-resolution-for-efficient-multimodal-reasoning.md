## 摘要
多模态大语言模型（MLLMs）通过提高输入保真度来获得更强的视觉理解能力，但由此产生的视觉 token 增长使得同时维持高空间分辨率和长时间上下文变得成本极高。我们认为，瓶颈不在于编码后的表示如何压缩，而在于编码器接收的像素量。为此，我们提出了 ResAdapt，这是一种输入端适配框架，用于学习在编码前每帧应分配多少视觉预算。ResAdapt 将轻量级分配器（Allocator）与未经修改的 MLLM 主干网络相结合，使主干网络在接收算子转换后的输入时，仍能保留其原生的视觉 token 接口。我们将分配过程建模为上下文老虎机（contextual bandit），并使用成本感知策略优化（CAPO）来训练分配器，该方法将稀疏的 rollout 反馈转化为稳定的准确度-成本学习信号。在预算受限的视频问答、时间定位和图像推理任务中，ResAdapt 改善了低预算运行点，且通常处于或接近效率-准确度前沿，在激进压缩下的推理密集型基准测试中提升最为显著。值得注意的是，在相同的视觉预算下，ResAdapt 支持多达 16 倍的帧数，同时带来超过 15% 的性能提升。

## Abstract
Multimodal Large Language Models (MLLMs) achieve stronger visual understanding by scaling input fidelity, yet the resulting visual token growth makes jointly sustaining high spatial resolution and long temporal context prohibitive. We argue that the bottleneck lies not in how post-encoding representations are compressed but in the volume of pixels the encoder receives, and address it with ResAdapt, an Input-side adaptation framework that learns how much visual budget each frame should receive before encoding. ResAdapt couples a lightweight Allocator with an unchanged MLLM backbone, so the backbone retains its native visual-token interface while receiving an operator-transformed input. We formulate allocation as a contextual bandit and train the Allocator with Cost-Aware Policy Optimization (CAPO), which converts sparse rollout feedback into a stable accuracy-cost learning signal. Across budget-controlled video QA, temporal grounding, and image reasoning tasks, ResAdapt improves low-budget operating points and often lies on or near the efficiency-accuracy frontier, with the clearest gains on reasoning-intensive benchmarks under aggressive compression. Notably, ResAdapt supports up to 16x more frames at the same visual budget while delivering over 15% performance gain. Code is available at https://github.com/Xnhyacinth/ResAdapt.

#### 2. 论文提出的方法论
*   **核心思想**：将每帧的视觉预算分配建模为一个“上下文老虎机（Contextual Bandit）”问题，根据文本查询（Query）和视频内容动态决定每帧的分辨率缩放比例。
*   **关键技术细节**：
    *   **分配器（Allocator）**：采用轻量级架构（如 SmolVLM），利用粗粒度视觉特征和查询文本，通过 Transformer 解码器生成每帧的隐藏状态。

    *   **成本感知策略优化（CAPO）**：一种强化学习算法，通过**动态成本枢轴（Dynamic Cost Pivot）**和**非对称奖励塑造（Asymmetric Reward Shaping）**来平衡准确率和成本，防止策略坍缩至最小预算。
    *   **正则化项**：引入**时间相似性损失（$L_{sim}$）**，惩罚相邻相似帧同时分配高预算的行为，强制模型学习差异化的感知策略。
*   **算法流程**：分配器预测缩放比例 -> 算子执行双线性插值调整图像大小 -> MLLM 主干网络单次前向推理 -> 根据回答准确度和 Token 成本计算奖励并更新参数。

#### 6. 论文的主要结论与发现
*   **效率飞跃**：ResAdapt 在消除 90% 视觉 Token 的情况下，性能仍能匹配或超过全量输入基线。
*   **时空转换红利**：在相同计算预算下，ResAdapt 可处理 **16 倍**更多的帧数，在复杂长视频任务上获得超过 15% 的相对增益。
*   **涌现的主动感知**：分配器学会了自动将像素集中在信息密集的关键帧（如字幕出现、场景切换、动作发生处），这种能力完全通过任务奖励学习得到，无需显式显著性标注。
*   **硬件友好**：由于不改变 Token 排布，ResAdapt 与 FlashAttention 和 vLLM 等推理加速引擎完全兼容。

#### 7. 优点
*   **干预点前移**：在编码前优化像素，从源头节省计算资源。
*   **即插即用**：训练好的分配器可以迁移到不同的 MLLM 主干网络上（如从 Qwen2.5 迁移至 Qwen3）。
*   **训练稳定性**：CAPO 解决了强化学习在处理准确度-成本权衡时极易坍缩的问题。
*   **任务感知**：分配策略是查询相关的（Query-aware），能根据问题重点调整视觉关注点。

#### 8. 不足与局限
*   **短视频开销**：分配器本身存在固定的计算开销，在短视频（如 $\le 32$ 帧）中，这种开销可能会抵消部分后端节省的成本。
*   **开环决策**：分配器在推理开始前就决定了预算，一旦分配错误（如过度压缩了关键帧），后续推理过程无法动态修正。
*   **跨模态泛化**：虽然在视频上表现优异，但在纯图像任务上的表现提升不明显，甚至在某些文本密集型图像任务中因分辨率降低而导致性能下降。

## 摘要
视觉几何 Transformer (VGGT) 是一种用于多种 3D 任务的强大前馈模型，但其交替注意力 (AA) 堆栈的计算量随总令牌数呈二次方增长，导致处理长视频序列的成本很高。现有的令牌减少加速器在 AA 内部运行，使得进入 AA 的补丁网格保持未压缩状态。我们提出了 PaceVGGT，这是一种 AA 前令牌剪枝框架，在冻结的 VGGT 的第一个 AA 块之前对 DINO 补丁令牌进行剪枝。PaceVGGT 训练了一个轻量级的令牌评分器，用于根据 DINO 特征估计每个令牌的重要性。该评分器首先针对来自未剪枝骨干网络的 AA 内部注意力目标进行蒸馏，然后在下游相机、深度和点图损失下进行微调。每帧保留预算固定了骨干网络可见的序列长度，而重要性自适应的合并/剪枝分配则在固定的总合并预算下保留了高显著性帧的残余内容。特征引导恢复模块重建了预测头所需的密集空间网格。在 ScanNet-50 和 7-Scenes 数据集上，PaceVGGT 在降低推理延迟的同时，保持在重建质量-延迟的前沿。在 ScanNet-50 上，当 N=300 时，它比未修改的 VGGT 减少了 5.1 倍的延迟；当 N=1000 时，比 LiteVGGT 减少了 1.47 倍。这些结果表明，AA 前剪枝是冻结的 VGGT 式几何 Transformer 的一种可行加速路径。

## Abstract
Visual Geometry Transformer (VGGT) is a strong feed-forward model for multiple 3D tasks, but its Alternating-Attention (AA) stack scales quadratically in the total token count, making long clips expensive.   Existing token-reduction accelerators operate inside AA, leaving the patch grid that enters AA uncompressed.   We introduce PaceVGGT, a pre-AA token pruning framework that prunes DINO patch tokens before the first AA block of a frozen VGGT. PaceVGGT trains a lightweight Token Scorer that estimates per-token importance from DINO features. The scorer is first distilled against an AA-internal attention target from the unpruned backbone, then refined under downstream camera, depth, and point-map losses.   A per-frame keep budget fixes the backbone-visible sequence length, while an importance-adaptive merge/prune assignment preserves residual content from high-saliency frames under a fixed total merge budget.   A Feature-guided Restoration module reconstructs the dense spatial grid required by the prediction heads.   On ScanNet-50 and 7-Scenes, PaceVGGT remains on the reconstruction quality--latency frontier while reducing inference latency.   On ScanNet-50, it reduces latency by \(5.1\times\) over unmodified VGGT at \(N=300\) and \(1.47\times\) over LiteVGGT at \(N=1000\).   These results identify pre-AA pruning as a viable acceleration route for frozen VGGT-style geometry transformers.

### 1. 核心问题与整体含义（研究动机和背景）
视觉几何 Transformer (VGGT) 是处理 3D 几何任务（如深度估计、相机位姿预测）的强大模型，但其核心的“交替注意力”（Alternating-Attention, AA）机制在处理长视频序列时面临严重的计算瓶颈。由于自注意力机制的复杂度随令牌（Token）数量呈二次方增长，长序列会导致显存溢出和推理缓慢。
**研究动机：** 现有的加速方案（如 LiteVGGT, FastVGGT）多在 AA 堆栈内部进行令牌合并或剪枝，这意味着模型的前几层仍需处理全量数据。本文提出 **PaceVGGT**，主张在数据进入 AA 堆栈之前的“源头”进行剪枝，从而让整个骨干网络都能享受计算减量带来的红利。

### 2. 方法论：核心思想与技术细节
PaceVGGT 的核心是在冻结的 VGGT 骨干网络外包裹三个轻量级模块：
*   **令牌评分器 (Token Scorer)：** 一个轻量级的深度卷积网络，直接作用于 DINO 提取的特征图。它通过学习来预测每个 Token 的“几何重要性”。
    *   **监督信号：** 采用蒸馏法，目标值结合了相机锚定得分（相机 Token 对补丁的注意力）和跨视图匹配得分（Token 间的全局匹配强度）。
*   **重要性自适应剪枝 (Token Pruning)：** 将 Token 分为“保留 (Keep)”、“合并 (Merge)”和“剔除 (Prune)”三类。
    *   **自适应分配：** 根据帧的重要性动态分配合并预算，高显著性帧保留更多细节，低显著性帧则大幅剪枝。
    *   **合并机制：** 将被合并 Token 的特征加权聚合到最近的保留 Token 中，以减少信息损失。
*   **特征引导恢复 (Feature-guided Restoration)：** 由于下游预测头需要密集的空间网格，该模块利用交叉注意力机制，以原始 DINO 特征为查询 (Query)，将骨干网络输出的稀疏特征还原为密集网格。

### 3. 实验设计
*   **数据集：** 主要在 **ScanNet-50** 上进行训练和评估，并在 **7-Scenes** 和 **NRGBD** 数据集上进行零样本 (Zero-shot) 迁移测试。
*   **Benchmark 与对比方法：**
    *   基准：原始 **VGGT**。
    *   对比加速方法：**FastVGGT**、**Co-Me**、**LiteVGGT**（均为在 AA 内部进行处理的 SOTA 方法）。
*   **评估指标：** 3D 重建质量（倒角距离 CD、准确率、完整性）、相机位姿精度（ATE、ARE）以及端到端推理延迟。

### 4. 资源与算力
*   **硬件：** 所有实验均在单张 **NVIDIA H100 GPU** 上运行，使用了 FP16 混合精度和 Flash-Attention 2。
*   **训练时长：** 令牌评分器和恢复模块的训练总计约 **200 GPU 小时**。
*   **推理开销：** 额外增加的模块参数量不到原始 VGGT 的 1%，计算开销极低。

### 5. 实验数量与充分性
*   **实验规模：** 涵盖了从 100 帧到 1000 帧的不同序列长度测试。
*   **消融实验：** 非常充分。作者验证了 3D 意识监督的必要性（对比了 DINO 相似度剪枝）、三路分配策略的有效性、两阶段训练方案的优越性，以及不同超参数（如保留比例 $r$、混合系数 $\alpha$）的影响。
*   **客观性：** 通过在多个未见数据集上进行零样本测试，证明了该方法具有良好的泛化能力，而非仅仅在特定数据集上过拟合。

### 6. 主要结论与发现
*   **显著加速：** 在 ScanNet-50 上，当序列长度 $N=300$ 时，PaceVGGT 比原始 VGGT 快 **5.1 倍**；当 $N=1000$ 时，比当前最强的 LiteVGGT 快 **1.47 倍**。
*   **质量保持：** 在大幅提速的同时，3D 重建和位姿估计的精度几乎没有下降，甚至在某些长序列场景下优于基准模型（可能由于剪枝起到了去噪作用）。
*   **关键发现：** 证明了在 AA 堆栈之前进行剪枝是可行的，前提是必须使用“几何感知”的评分准则，而非简单的 2D 特征相似度。

### 7. 优点
*   **架构优势：** “源头剪枝”策略比“中途剪枝”更彻底，能最大限度减少计算量和显存占用。
*   **几何感知：** 通过蒸馏 AA 内部的注意力图，成功捕捉到了对 3D 重建至关重要的特征（如边缘、角点、跨视图对应点）。
*   **即插即用：** 保持 VGGT 骨干网络冻结，仅训练极小规模的辅助模块，易于集成。

### 8. 不足与局限
*   **领域偏移风险：** 评分器在 ScanNet 上训练，若遇到极端的户外场景或大基线运动，评分准确性可能会下降。
*   **恢复模块限制：** 目前仅使用单层交叉注意力进行特征恢复，在极高分辨率输出需求下，可能存在表达能力不足的问题。
*   **任务覆盖：** 论文未评估 2D 跟踪（Tracking）任务的性能，仅关注了重建和位姿。
*   **超参数敏感性：** 针对不同场景（如室内 vs 室外），可能需要手动调整保留比例和混合系数。

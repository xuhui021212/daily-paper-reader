## 摘要
自回归视频扩散模型支持实时合成，但在长时程生成中面临误差累积和上下文丢失的问题。我们发现，自回归视频扩散 Transformer 中的注意力头在功能上扮演着不同的角色：用于细节细化的局部头、用于结构稳定的锚点头以及用于长程上下文聚合的记忆头。然而，现有方法对它们进行统一处理，导致 KV 缓存分配效率低下。我们提出了 Head Forcing，这是一个无需训练的框架，为每种类型的注意力头分配定制的 KV 缓存策略：局部头和锚点头仅保留必要的 token，而记忆头则采用具有动态情节更新的层次化记忆系统，以确保长程一致性。此外，一种按头进行的 RoPE 重编码方案进一步确保了位置编码保持在预训练范围内。在无需额外训练的情况下，Head Forcing 将生成时长从 5 秒延长至分钟级，支持多提示词交互式合成，并持续优于现有基准。项目主页：https://jiahaotian-sjtu.github.io/headforcing.github.io/。

## Abstract
Autoregressive video diffusion models support real-time synthesis but suffer from error accumulation and context loss over long horizons. We discover that attention heads in AR video diffusion transformers serve functionally distinct roles as local heads for detail refinement, anchor heads for structural stabilization, and memory heads for long-range context aggregation, yet existing methods treat them uniformly, leading to suboptimal KV cache allocation. We propose Head Forcing, a training-free framework that assigns each head type a tailored KV cache strategy: local and anchor heads retain only essential tokens, while memory heads employ a hierarchical memory system with dynamic episodic updates for long-range consistency. A head-wise RoPE re-encoding scheme further ensures positional encodings remain within the pretrained range. Without additional training, Head Forcing extends generation from 5 seconds to minute-level duration, supports multi-prompt interactive synthesis, and consistently outperforms existing baselines. ### 1. 核心问题与整体含义（研究动机和背景）
自回归（AR）视频扩散模型虽然支持实时流式合成，但在生成长视频时面临两大挑战：**误差累积**（导致颜色漂移和细节丢失）和**上下文丢失**（由于 KV 缓存容量限制导致身份漂移和场景遗忘）。
作者发现，现有方法对 Transformer 中的所有注意力头采取统一的 KV 缓存策略是低效的。通过分析发现，注意力头具有**异质性**，可分为：
*   **局部头 (Local Heads)**：关注当前帧及邻近帧，负责细节细化。
*   **锚点头 (Anchor Heads)**：强力关注初始帧，负责结构稳定。
*   **记忆头 (Memory Heads)**：广泛关注历史上下文，负责长程一致性。
**研究动机**：利用这种头部的功能差异，设计一种无需训练的框架，通过精细化管理 KV 缓存来实现稳定、长时、可控的视频生成。

### 2. 方法论：核心思想与关键技术
Head Forcing 是一个**无需训练 (Training-free)** 的框架，核心技术包括：

*   **离线分析 (Head Profiling)**：通过计算注意力权重在不同时间桶（起始桶、中间桶、当前桶）的分布，将预训练模型的注意力头分类为局部、锚点和记忆头。
*   **按头分配 KV 缓存 (Head-wise KV Cache Allocation)**：
    *   **局部头**：仅保留最近的 1 帧和当前块，减少噪声传播。
    *   **锚点头**：保留前 3 帧作为“锚点”以及最近帧，防止视觉崩溃。
    *   **记忆头**：采用**层次化记忆系统**。包括“快速记忆”（邻近场景）和“情节记忆”（存储代表性帧）。
*   **动态情节更新与压缩**：
    *   **准入机制**：基于 Key 嵌入的余弦相似度计算“场景新颖度”，仅在出现新场景时更新情节记忆。
    *   **提示词引导压缩**：当记忆满时，根据当前文本提示词的语义相关性，对冗余 token 进行重要性筛选和压缩。
*   **头级 RoPE 重编码 (Head-wise RoPE Re-encoding)**：针对不同头部的缓存长度，重新映射时间位置索引，确保相对位置编码始终处于预训练范围内，消除位置 O.O.D. 导致的伪影。

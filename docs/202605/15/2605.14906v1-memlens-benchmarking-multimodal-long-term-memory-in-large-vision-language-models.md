## 摘要
记忆对于大型视觉语言模型（LVLMs）处理长程多模态交互至关重要，目前有两种技术路径提供这种能力：长上下文 LVLMs 和记忆增强型智能体。然而，现有的基准测试尚未在真正需要多模态证据的问题上对两者进行系统性比较。为了填补这一空白，我们推出了 MEMLENS，这是一个针对多模态多轮会话记忆的全面基准测试，包含 789 个问题，涵盖五种记忆能力（信息提取、多轮推理、时间推理、知识更新和拒绝回答），并在跨模态 Token 计数方案下设定了四个标准上下文长度（32K-256K tokens）。一项图像消融研究证实，解决 MEMLENS 需要视觉证据：在 80.4% 的证据包含图像的问题中，移除证据图像会导致两个前沿 LVLMs 的准确率降至 2% 以下。通过对 27 个 LVLMs 和 7 个记忆增强型智能体的评估，我们发现长上下文 LVLMs 通过直接的视觉定位在短上下文中实现了高准确率，但随着对话增长性能会下降；而记忆智能体在长度上表现稳定，但在存储压缩时会丢失视觉保真度。多轮推理使大多数系统的表现限制在 30% 以下，且没有任何一种方法能单独解决该任务。这些结果促使人们开发结合了长上下文注意力和结构化多模态检索的混合架构。我们的代码可在 https://github.com/xrenaf/MEMLENS 获取。

## Abstract
Memory is essential for large vision-language models (LVLMs) to handle long, multimodal interactions, with two method directions providing this capability: long-context LVLMs and memory-augmented agents. However, no existing benchmark conducts a systematic comparison of the two on questions that genuinely require multimodal evidence. To close this gap, we introduce MEMLENS, a comprehensive benchmark for memory in multimodal multi-session conversations, comprising 789 questions across five memory abilities (information extraction, multi-session reasoning, temporal reasoning, knowledge update, and answer refusal) at four standard context lengths (32K-256K tokens) under a cross-modal token-counting scheme. An image-ablation study confirms that solving MEMLENS requires visual evidence: removing evidence images drops two frontier LVLMs below 2% accuracy on the 80.4% of questions whose evidence includes images. Evaluating 27 LVLMs and 7 memory-augmented agents, we find that long-context LVLMs achieve high short-context accuracy through direct visual grounding but degrade as conversations grow, whereas memory agents are length-stable but lose visual fidelity under storage-time compression. Multi-session reasoning caps most systems below 30%, and neither approach alone solves the task. These results motivate hybrid architectures that combine long-context attention with structured multimodal retrieval. Our code is available at https://github.com/xrenaf/MEMLENS.

#### 1. 核心问题与研究动机
论文关注的核心问题是：**如何系统性地评估和比较大型视觉语言模型（LVLMs）在长程多模态对话中的长期记忆能力？**
目前提升 LVLM 记忆能力主要有两条路径：一是扩展原生上下文窗口（长上下文 LVLMs），二是通过外部存储进行检索增强（记忆增强型智能体）。然而，现有基准测试存在以下不足：
*   缺乏对这两类技术路径在相同标准下的直接对比。
*   现有基准中的问题往往可以通过纯文本“走捷径”回答，并不真正依赖视觉证据。
*   缺乏对上下文长度（从 32K 到 256K）的标准化控制。

#### 2. 方法论：MEMLENS 基准设计
MEMLENS 是一个包含 789 个问题的综合基准，其核心思想是通过“实体抽象”强制模型必须结合视觉和文本证据。
*   **五大记忆能力：**
    1.  **信息提取 (IE)：** 从单次会话中检索视觉细节。
    2.  **多会话推理 (MSR)：** 跨 3-8 个会话进行计数、算术或实体消解。
    3.  **时间推理 (TR)：** 结合时间戳和视觉线索（如时钟、日历）判断时长或排序。
    4.  **知识更新 (KU)：** 追踪用户偏好的动态变化（如从喜欢苹果变为喜欢猕猴桃）。
    5.  **拒绝回答 (AR)：** 当证据被移除时，模型应拒绝回答而非幻觉。
*   **构建流程：**
    1.  **多模态会话模拟：** 基于主题本体从 Web 检索图像，利用 GPT-5.1 和 Gemini-3-Pro 模拟多轮对话。
    2.  **实体抽象 (关键技术)：** 将文本中的具体实体（如“金门大桥”）替换为抽象代词（如“图像中的这座桥”），迫使模型必须查看图像才能确定指代对象。
    3.  **对话历史组装：** 将证据会话插入由无关“填充会话”组成的背景中，通过跨模态 Token 计数方案精确控制上下文长度（32K, 64K, 128K, 256K）。

#### 3. 实验设计
*   **评估对象：** 共评估了 34 个系统。
    *   **27 个 LVLMs：** 包括闭源模型（GPT-5.4, Gemini-3.1-Pro, Claude 4.5）和开源模型（Kimi-K2.5, Qwen3.5, GLM-4.6V, Gemma3 等）。

*   **算力设备：** 本地模型使用 8 张 NVIDIA A100-80GB 显卡进行推理，采用 vLLM 框架和张量并行技术处理 128K 及以上上下文。

*   **实验规模：** 涵盖了 789 个精心设计的问题，并在 4 个长度梯度上对 34 个模型进行了全面测试，总计产生数万次模型调用。

    *   采用了三轮人工审核（问题级、会话级、填充内容级）。

    *   实验设计充分考虑了统计显著性，并进行了错误类型深度分析。

#### 6. 主要结论与发现
*   **能力独立性：** 五种记忆能力之间相关性较低，强信息提取能力不代表强多会话推理能力。
*   **技术路径对比：**
    *   **长上下文 LVLMs：** 短上下文表现优异，但随长度增加性能显著下降（尤其是拒绝回答能力）。
    *   **记忆智能体：** 性能随长度变化稳定，但由于存储时的压缩（如转为 Caption 或 Embedding），会丢失细粒度视觉信息，导致视觉保真度不足。
*   **性能瓶颈：** 多会话推理（MSR）是所有系统的天花板，大多数模型准确率低于 30%。
*   **错误根源：** 信息提取错误中约 90% 源于视觉感知/检索失败，而非推理逻辑错误。

#### 7. 论文优点
*   **视觉必要性强：** 通过实体抽象技术解决了以往基准中“视觉冗余”的问题。
*   **标准化程度高：** 首次在统一的长度控制和跨模态计数方案下对比了 LVLM 和记忆智能体。
*   **分析深入：** 不仅给出了分数，还通过错误分类、检索归因等手段揭示了不同架构的互补性缺陷。

#### 8. 不足与局限
*   **合成数据偏差：** 尽管经过人工审核，但对话仍由 LLM 生成，可能与真实人类长程交互的分布存在差异。
*   **裁判模型偏见：** 尽管进行了验证，但 LLM 裁判在处理极短答案或特定格式时仍表现出一定的宽容度（Leniency）。
*   **静态评估：** 采用的是离线查询模式，未能完全模拟在线流式交互中的实时更新和遗忘动态。

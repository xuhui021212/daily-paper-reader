## 摘要
视频大多模态模型日益面临可扩展性瓶颈：长视频会产生过长的视觉标记（visual-token）序列，这在推理过程中急剧增加了内存消耗和延迟。虽然现有的压缩方法在特定设置下有效，但大多数方法要么对查询的感知较弱，要么在所有帧上应用固定的压缩策略，当视觉证据在时间上分布不均时，这些方法被证明是次优的。为了解决这一问题，我们提出了 VideoRouter，这是一个基于 InternVL 构建的查询自适应双路由器框架，用于预算受限下的证据分配。语义路由器（Semantic Router）预测主导分配策略，在广泛的时间覆盖和自适应高分辨率保留之间进行选择；而图像路由器（Image Router）则利用早期大语言模型（LLM）层对帧相关性进行评分。这使得能够对相关性较低的帧进行激进压缩，同时保留关键证据帧的细节。为了训练这两个路由器，我们构建了用于分配策略监督的 Video-QTR-10K 数据集和用于帧相关性监督的 Video-FLR-200K 数据集。在 VideoMME、MLVU 和 LongVideoBench 上的实验表明，在相当或更低的预算下，VideoRouter 相比 InternVL 基准模型持续提升了性能，并实现了高达 67.9% 的标记缩减。

## Abstract
Video large multimodal models increasingly face a scalability bottleneck: long videos produce excessively long visual-token sequences, which sharply increase memory and latency during inference. While existing compression methods are effective in specific settings, most are either weakly query-aware or apply a fixed compression policy across frames, proving suboptimal when visual evidence is unevenly distributed over time. To address this, we present VideoRouter, a query-adaptive dual-router framework built on InternVL for budgeted evidence allocation. The Semantic Router predicts the dominant allocation policy, choosing between broad temporal coverage and adaptive high-resolution preservation, while the Image Router uses early LLM layers to score frame relevance. This enables aggressive compression on less relevant frames while preserving detail on critical evidence frames. To train both routers, we build Video-QTR-10K for allocation-policy supervision and Video-FLR-200K for frame-relevance supervision. Experiments on VideoMME, MLVU, and LongVideoBench show that VideoRouter consistently improves over the InternVL baseline under comparable or lower budgets, achieving up to a 67.9% token reduction.

长视频理解在多模态大模型（MLLM）中面临严重的**扩展性瓶颈**：长视频生成的视觉标记（Visual Tokens）数量庞大，导致推理时的内存占用和延迟剧增。
*   **现有痛点**：目前的压缩方法多采用固定策略（如每帧统一压缩）或弱查询感知，忽略了视频中证据分布的**非均匀性**。
*   **核心理念**：论文将长视频压缩建模为“**受限预算下的证据分配**”问题。即在有限的 Token 预算内，模型应根据问题动态决定是进行“全局覆盖”还是“局部精细化保留”。

### 2. 方法论：核心思想与技术细节
VideoRouter 提出了一个基于 InternVL 的**查询自适应双路由框架**：

*   **双路由架构**：
    *   **语义路由器 (Semantic Router)**：在策略层面工作。通过分析问题文本，预测主导分配策略：是选择“全局策略”（Global，保持全视频粗略覆盖）还是“片段策略”（Fragment，对关键帧进行非均匀高分辨率分配）。
    *   **图像路由器 (Image Router)**：在帧层面工作。复用 LLM 的前 4 层（Shadow Stack）作为特征提取器，计算每一帧与查询的相关性得分。
*   **动态预算分配算法**：
    *   根据 LLM 上下文窗口动态计算可用视觉 Token 预算 $B_{img}$。
    *   **优先级池化**：若为片段策略，优先保证高相关性帧以高分辨率（Scale=1）保留，不相关帧则进行激进压缩（Scale=4）或丢弃，确保不超出预算。
*   **监督数据构建**：
    *   **Video-FLR-200K**：包含 20 万个 QA 对，带有帧级相关性标注，用于训练图像路由器。
    *   **Video-QTR-10K**：包含 1 万个视频-查询对，带有分配策略标注，用于训练语义路由器。
*   **训练流程**：采用三阶段训练：1. 图像路由器预训练；2. 语义路由器训练；3. 联合微调（对齐压缩序列与答案生成）。

*   **实验规模**：涵盖了 5 个大型基准测试，并进行了详尽的消融实验。

    *   **消融实验**：分别验证了语义路由器、图像路由器以及联合微调阶段的必要性。

### 6. 主要结论与发现
*   **高效性**：VideoRouter 在显著减少 Token 数量（最高减少 **67.9%**）的情况下，性能依然优于或持平于全量 Token 的基准模型。
*   **性能提升**：在 LongVideoBench 等长视频任务上，相比 InternVL3 基准有明显提升（如在 12K 预算下，准确率从 56.9% 提升至 61.9%）。
*   **路由深度**：发现复用 LLM 的前 4 层作为路由器特征提取器是性能与延迟的最佳平衡点。

### 7. 优点（亮点）
*   **查询自适应**：不再死板地压缩所有帧，而是根据“问题需要什么”来决定“保留什么”。
*   **轻量化路由**：复用 LLM 早期层，避免了引入沉重的额外参数，且降低了推理延迟（TTFT）。
*   **高质量数据集**：构建了专门用于路由决策的监督数据集，填补了该领域训练数据的空白。
*   **灵活的预算控制**：能够严格遵守给定的 Token 预算，避免 OOM（显存溢出）。

### 8. 不足与局限
*   **离线假设**：目前的实现假设可以离线访问所有采样帧，对于实时流媒体视频的在线处理支持尚不明确。
*   **教师模型偏差**：监督数据是由更强的 VLM/LLM 自动生成的，可能存在教师模型的固有偏差或标注噪声。
*   **评估范围**：虽然在 8B 规模模型上表现优异，但在更大规模（如 70B+）模型上的扩展性表现有待进一步验证。

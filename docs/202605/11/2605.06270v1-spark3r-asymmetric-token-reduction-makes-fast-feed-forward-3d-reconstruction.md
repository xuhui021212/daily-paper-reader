## 摘要
基于 Vision Transformer 的前馈三维重建模型可以直接从少量输入图像中估计场景几何和相机位姿，但由于全局注意力层的二次计算开销，将其扩展到包含数百或数千帧的视频输入仍然具有挑战性。最近的 Token 合并方法通过在全局注意力层内压缩 Token 序列来加速这些模型，但它们对查询（query）Token 和键值（key-value）Token 应用统一的缩减，忽略了它们在三维重建中功能上的不同角色。在这项工作中，我们发现了前馈三维重建模型的一个关键特性：查询 Token 编码了特定视角的几何请求且对压缩敏感，而键值 Token 代表共享的场景上下文，能够容忍激进的压缩。基于这一见解，我们提出了 Spark3R，这是一个无需训练的加速框架，它通过分配不同的缩减因子来解耦查询 Token 和键值 Token 的压缩，对查询 Token 应用组内 Token 合并，对键值 Token 应用轻量级 Token 剪枝。此外，Spark3R 跨层自适应地调整键值缩减因子，进一步优化了质量与效率之间的权衡。作为一个无需重新训练的即插即用框架，Spark3R 直接集成到多个预训练的前馈三维重建模型中，包括 VGGT、π³ 和 Depth-Anything-3，在 1,000 帧输入上实现了高达 28 倍的加速，同时保持了具有竞争力的重建质量。

## Abstract
Feed-forward 3D reconstruction models based on Vision Transformers can directly estimate scene geometry and camera poses from a small set of input images, but scaling them to video inputs with hundreds or thousands of frames remains challenging due to the quadratic cost of global attention layers. Recent token-merging methods accelerate these models by compressing the token sequence within the global attention layers, but they apply a uniform reduction to query tokens and key-value tokens, ignoring their functionally distinct roles in 3D reconstruction. In this work, we identify a key property of feed-forward 3D reconstruction models: query tokens encode view-specific geometric requests and are sensitive to compression, while key-value tokens represent shared scene context and tolerate aggressive compression. Guided by this insight, we propose Spark3R, a training-free acceleration framework that decouples the compression of query tokens and key-value tokens by assigning distinct reduction factors, with intra-group token merging applied to query tokens and lightweight token pruning to key-value tokens. Additionally, Spark3R adaptively adjusts the key-value reduction factor across layers, further improving the quality-efficiency trade-off. As a plug-and-play framework requiring no retraining, Spark3R integrates directly into multiple pretrained feed-forward 3D reconstruction models, including VGGT, $π^3$, and Depth-Anything-3, and achieves up to $28\times$ speedup on 1,000-frame inputs while maintaining competitive reconstruction quality.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：基于 Vision Transformer (ViT) 的前馈三维重建模型（如 VGGT, $\pi^3$）虽然能直接从多视图预测几何和位姿，但其全局注意力机制的计算开销随帧数呈**二次方增长**。这导致模型难以处理包含数百或数千帧的长视频。
*   **研究动机**：现有的 Token 压缩方法（如 ToMe）对查询（Query, Q）和键值（Key-Value, KV）标记采用统一的缩减比例。作者发现 Q 和 KV 在重建中角色不同：**Q 编码特定视角的几何请求（对压缩极敏感），而 KV 代表共享的场景上下文（对压缩极具容忍度）**。现有的统一压缩策略未能利用这种非对称性，限制了加速比和重建质量。

### 2. 方法论
Spark3R 是一个**无需训练、即插即用**的加速框架，包含三个核心设计：
*   **非对称 Token 缩减 (Asymmetric Token Reduction)**：为 Q 和 KV 分配独立的缩减因子 $r_Q$ 和 $r_{KV}$。通常 $r_{KV} \gg r_Q$，以在保留视角细节的同时大幅压缩背景上下文。
*   **组内查询合并 (Intra-Group Query Merging)**：针对 Q 标记，利用“合并对通常具有时间局部性”的发现，将帧划分为大小为 $G$ 的组（默认 $G=20$），仅在组内进行 Token 合并。这把匹配复杂度从 $O(S^2)$ 降低到 $O(S \cdot G)$（$S$ 为帧数）。
*   **轻量级键值剪枝 (Lightweight KV Pruning)**：由于 KV 标记之间相似度极高（>0.9），作者将复杂的“合并-平均”操作简化为“直接剪枝”，通过时间步长采样丢弃冗余 Token，几乎不产生额外计算开销。
*   **层自适应 KV 缩减方案 (Layer-Adaptive Schedule)**：通过离线探测发现模型中后部层对压缩更敏感。框架据此将层分为高/低敏感两类，对低敏感层应用更激进的压缩（如 3 倍基础缩减率）。

### 6 主要结论与发现
*   **极致加速**：在 1000 帧输入下，Spark3R 实现了高达 **28 倍** 的推理加速。

*   **通用性**：作为即插即用框架，它在不同架构（交替注意力、全等变架构、纯 ViT）上均表现稳健。

### 7. 优点（亮点）
*   **非对称视角**：首次深入分析并利用了 Q 和 KV 在 3D 重建任务中的功能差异。
*   **零成本集成**：无需任何重新训练或微调，极大降低了工业界部署长视频重建模型的门槛。
*   **计算效率极高**：通过组内合并和直接剪枝，将 Token 缩减本身的开销降至最低。

### 8. 不足与局限
*   **性能上限受限**：作为无需训练的方法，其性能上限始终受限于基础预训练模型。如果基础模型本身在某些场景失效，Spark3R 无法通过加速来修复其内在缺陷。
*   **离线探测需求**：层自适应方案需要对每个新模型进行一次简单的离线敏感度探测（虽然只需一次且很快）。
*   **未来改进空间**：作者提到，如果能结合联合微调（Joint Fine-tuning），可能会进一步突破目前的质量瓶颈。

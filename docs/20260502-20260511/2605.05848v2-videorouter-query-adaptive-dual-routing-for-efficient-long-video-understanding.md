## 摘要
视频大多模态模型日益面临可扩展性瓶颈：长视频会产生过长的视觉 Token 序列，从而在推理过程中急剧增加内存消耗和延迟。虽然现有的压缩方法在特定设置下有效，但大多数方法要么对查询的感知较弱，要么对所有帧应用固定的压缩策略，当视觉证据在时间上分布不均时，这些方法表现不佳。为了解决这一问题，我们提出了 VideoRouter，这是一个基于 InternVL 构建的查询自适应双路由器框架，用于预算受限下的证据分配。语义路由器（Semantic Router）预测主导分配策略，在广泛的时间覆盖和自适应高分辨率保留之间进行选择；而图像路由器（Image Router）则利用 LLM 的早期层对帧相关性进行评分。这使得能够对相关性较低的帧进行激进压缩，同时保留关键证据帧的细节。为了训练这两个路由器，我们构建了用于分配策略监督的 Video-QTR-10K 数据集和用于帧相关性监督的 Video-FLR-200K 数据集。在 VideoMME、MLVU 和 LongVideoBench 上的实验表明，在相当或更低的预算下，VideoRouter 相比 InternVL 基线模型持续提升了性能，并实现了高达 67.9% 的 Token 削减。

## Abstract
Video large multimodal models increasingly face a scalability bottleneck: long videos produce excessively long visual-token sequences, which sharply increase memory and latency during inference. While existing compression methods are effective in specific settings, most are either weakly query-aware or apply a fixed compression policy across frames, proving suboptimal when visual evidence is unevenly distributed over time. To address this, we present VideoRouter, a query-adaptive dual-router framework built on InternVL for budgeted evidence allocation. The Semantic Router predicts the dominant allocation policy, choosing between broad temporal coverage and adaptive high-resolution preservation, while the Image Router uses early LLM layers to score frame relevance. This enables aggressive compression on less relevant frames while preserving detail on critical evidence frames. To train both routers, we build Video-QTR-10K for allocation-policy supervision and Video-FLR-200K for frame-relevance supervision. Experiments on VideoMME, MLVU, and LongVideoBench show that VideoRouter consistently improves over the InternVL baseline under comparable or lower budgets, achieving up to a 67.9% token reduction.

    *   长视频中的关键证据分布极不均匀：有些问题需要全局覆盖（如剧情梗概），有些则依赖局部细节（如某个动作）。
    *   作者提出将长视频压缩视为一个**预算受限下的证据分配问题**，即在有限的 Token 预算内，动态决定是“撒大网”还是“精定位”。

### 2. 方法论
VideoRouter 框架基于 InternVL 构建，核心思想是**查询自适应的双路由机制**：
*   **语义路由器 (Semantic Router)**：
    *   **功能**：分析用户问题的语义，预测主导分配策略。
    *   **策略分类**：选择“全局策略”（Global，均匀粗略覆盖全视频）或“片段策略”（Fragment，针对特定证据帧保留高分辨率）。
*   **图像路由器 (Image Router)**：
    *   **功能**：当选择片段策略时，对每一帧进行相关性评分。
    *   **技术细节**：复用 LLM 的前 4 层作为“影子栈”（Shadow Stack）提取多模态融合特征，通过轻量级 MLP 预测帧与问题的相关性。
*   **动态预算分配算法**：
    *   **公式**：$B_{img} = L_{max} - L_{text} - L_{gen} - \epsilon$。根据 LLM 最大上下文长度减去文本和生成预留空间，动态计算视觉 Token 预算。
    *   **优先级分配**：优先保证高相关性帧的高分辨率（Scale=1），剩余预算分配给低相关性帧进行激进压缩（Scale=4），若预算仍不足则进行战略性丢帧。
*   **监督数据集构建**：
    *   **Video-FLR-200K**：提供 20 万对帧级相关性标注。
    *   **Video-QTR-10K**：提供 1 万条关于分配策略（全局 vs 片段）的监督信号。

### 3. 实验设计
*   **数据集/场景**：
    *   **Video-MME**：涵盖短、中、长视频（最长 1 小时）的综合评估。
    *   **LongVideoBench**：专注于长上下文交错视频理解。
    *   **MLVU、LVBench、EgoSchema**：评估细粒度时空理解和长视频推理。
*   **Benchmark 与对比方法**：
    *   **基线模型**：InternVL3-8B, Qwen2.5-VL。
    *   **对比压缩方法**：FastV, PruMerge, DyToK, QTSplus, DToMA, DIG 等。
    *   **控制变量实验**：在相同的 Token 预算（如 12,288）下，对比不同分配策略的性能。

### 4. 资源与算力
*   **硬件环境**：使用了 **4 张 NVIDIA H20 GPU**。
*   **训练细节**：
    *   采用三阶段训练：图像路由器预训练、语义路由器训练、最后进行联合微调（Joint Fine-Tuning）。

    *   使用了 DeepSpeed ZeRO-2 优化器。

### 5. 实验数量与充分性
*   **实验规模**：
    *   在 5 个主流长视频基准测试上进行了完整评估。
    *   进行了详尽的**消融实验**，验证了双路由器的各自贡献、LLM 早期层提取深度的影响（k=4 为最优平衡点）。
    *   进行了**跨架构验证**，将 VideoRouter 移植到 Qwen2.5-VL 上，证明了方法的通用性。
*   **客观性与公平性**：作者特别设计了“受控对比协议”，确保所有方法在相同的输入帧数和 Token 预算下竞争，排除了因 Token 数量不同带来的性能偏差。

### 6. 主要结论与发现
*   **性能提升**：VideoRouter 在显著减少 Token 的情况下，性能优于 InternVL 基线。例如在 Video-MME 上，以更少的 Token 实现了更高的准确率。
*   **高效压缩**：最高实现了 **67.9% 的 Token 削减**，同时降低了首字响应时间（TTFT）和峰值内存占用。
*   **策略有效性**：语义路由器预测准确率达 98.2%，图像路由器相关性评分准确率达 83.4%，证明了双路由机制能精准捕捉视觉证据。

### 7. 优点
*   **查询自适应**：不再盲目压缩，而是根据问题动态调整资源，符合人类观察视频的逻辑。
*   **轻量化设计**：复用 LLM 早期层作为特征提取器，避免了引入沉重的额外参数。
*   **预算感知**：显式地在 LLM 上下文窗口限制内优化分配，具有极强的工程实用性。
*   **高质量数据**：构建并开源了专门用于路由决策监督的大规模数据集。

### 8. 不足与局限
*   **模型规模限制**：主要在 8B 参数规模的模型上验证，尚未在超大规模模型（如 70B+）上广泛测试。
*   **离线处理**：目前依赖于对预采样帧的离线处理，对于流式长视频的实时处理能力尚待探讨。
*   **数据偏见风险**：监督数据由 VLM/LLM 辅助生成，可能继承了教师模型的固有偏见或错误。
*   **极端情况**：在极低预算下，虽然优于基线，但仍不可避免地会丢失部分细粒度空间信息。

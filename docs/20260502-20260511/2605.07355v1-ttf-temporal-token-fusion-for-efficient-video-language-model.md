## 摘要
随着视觉标记（token）数量随视频长度增加，视频语言模型（VLM）面临着快速增长的推理成本。例如，在 Qwen3-VL 中，448x448 分辨率下的 32 帧视频已产生超过 8,000 个视觉标记，使得大语言模型（LLM）的预填充（prefill）阶段成为主要的吞吐量瓶颈。现有方法通常依赖于全局相似性或注意力引导的压缩，这往往会抵消其带来的收益。我们提出了时间标记融合（Temporal Token Fusion, TTF），这是一种无需训练、即插即用的 LLM 前置标记压缩框架，利用了视频中的结构化时间冗余。TTF 自动选择一个锚点帧，然后对后续的每一帧执行局部窗口相似性搜索（例如 3x3），并融合超过阈值的标记。压缩后的序列通过坐标重对齐在预填充和解码阶段保持位置一致性，从而实现与现有 VLM 流水线的无缝集成。在阈值 t=0.70 的 Qwen3-VL-8B 上，TTF 移除了约 67% 的视觉标记，同时保留了 99.5% 的基准准确度，且仅引入了约 0.16 GFLOPs 的匹配开销。总之，TTF 为视频理解提供了一种实用且高效的解决方案。代码可在 https://github.com/Cominder/ttf 获取。

## Abstract
Video-language models (VLMs) face rapid inference costs as visual token counts scale with video length. For example, 32 frames at $448{\times}448$ resolution already yield >8,000 visual tokens in Qwen3-VL, making LLM prefill the dominant throughput bottleneck. Existing methods often rely on global similarity or attention-guided compression, incurring offsets to their gains. We propose \textbf{Temporal Token Fusion (TTF)}, a training-free, plug-and-play pre-LLM token compression framework that exploits structured temporal redundancy in video. TTF automatically selects an anchor frame, then for each subsequent frame, performs a local window similarity search (e.g.,$3\times 3$), fusing tokens that exceed a threshold. The compressed sequence maintains positional consistency across both prefill and decoding through coordinate realignment, enabling seamless integration with existing VLM pipelines. On Qwen3-VL-8B with threshold t=0.70, TTF removes about 67\% of visual tokens while retaining 99.5\% of the baseline accuracy and introducing only ${\approx}0.16$\,GFLOPs of matching overhead. Overall, TTF offers a practical, efficient solution for video understanding. The code is available at \href{https://github.com/Cominder/ttf}{https://github.com/Cominder/ttf}

### 1. 核心问题与整体含义
*   **研究动机**：视频语言模型（VLM）在处理长视频时，视觉标记（token）数量随帧数激增（如 Qwen3-VL 处理 32 帧可产生 >8000 个标记），导致 LLM 的预填充（prefill）阶段成为严重的计算瓶颈，并大幅增加 KV 缓存负担。
*   **核心挑战**：现有的压缩方法要么在 LLM 内部剪枝（与 FlashAttention 等高效算子不兼容），要么采用全局相似度匹配（计算复杂度随分辨率平方增长，开销巨大）。
*   **整体含义**：本文提出了一种名为 **TTF（时间标记融合）** 的方法，旨在通过利用视频在时间维度上的局部冗余，在进入 LLM 之前以极低的计算成本压缩视觉标记，实现高效且无损的视频理解。

### 2. 方法论
*   **核心思想**：自然视频具有点对点的时间冗余性，即同一场景区域在相邻帧的网格位置通常保持稳定或仅有微小偏移。
*   **关键技术细节**：
    1.  **锚点帧选择（Anchor Selection）**：计算每一帧平均特征与全视频全局平均特征的余弦相似度，选择最具有代表性的帧作为“锚点”。
    2.  **局部窗口搜索（Local Window Search）**：不进行全局匹配，而是将源帧标记与锚点帧中对应的 $r \times r$（默认 3x3）局部窗口内的标记进行相似度对比。
    3.  **阈值门控与融合**：设置相似度阈值 $t$。若源标记与锚点标记相似度超过 $t$，则视为冗余并融合（丢弃源标记，仅保留锚点）；否则保留。
    4.  **位置对齐（M-RoPE Alignment）**：压缩后，利用 Qwen3-VL 的多模态旋转位置嵌入（M-RoPE），确保保留的标记维持其原始的 $(k, y, x)$ 坐标，并在解码阶段通过坐标重对齐防止位置漂移导致的生成重复。
*   **算法复杂度**：复杂度为 $O((2r+1)^2 FNC)$，相对于标记数量呈线性增长，远低于传统全局匹配的 $O(FN^2C)$。

### 3. 实验设计
*   **数据集/场景**：
    *   **VideoMME**：综合性多模态视频评估基准。
    *   **MVBench**：涵盖 20 种细粒度时间理解任务。
    *   **MLVU**：多任务长视频理解。
    *   **LongVideoBench**：长上下文交错视频理解。
*   **对比方法**：包括原始基准（Baseline）、FastVID、VisionZip、HoliTom 和 VidCom 2。
*   **测试模型**：Qwen3-VL-2B-Instruct 和 Qwen3-VL-8B-Instruct。

### 4. 资源与算力
*   **硬件环境**：所有实验均在单张 **NVIDIA A100 40GB GPU** 上完成。
*   **训练时长**：该方法为 **无需训练（Training-free）** 的插件，因此无训练算力消耗。
*   **推理开销**：TTF 引入的匹配开销极低，仅约 **0.16 GFLOPs**，在端到端推理中几乎可以忽略不计。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 4 个主流基准测试，对比了 5 种不同的压缩策略，并针对 Qwen3-VL 的两个参数规模进行了验证。
*   **消融实验**：详细探讨了相似度阈值 $t$、锚点帧选择策略（首帧、末帧、自动选择）以及局部搜索窗口大小（1x1, 3x3, 5x5）对性能的影响。
*   **充分性评价**：实验设计较为全面，不仅关注准确率，还详细记录了延迟、吞吐量和显存占用。对比实验在相似的压缩率下进行，保证了公平性。

### 6. 主要结论与发现
*   **高效压缩**：在 Qwen3-VL-8B 上，当 $t=0.70$ 时，TTF 移除了约 **67%** 的视觉标记，而准确率保留率高达 **99.5%**。
*   **性能提升**：在某些情况下（如 8B 模型），压缩后的模型在 VideoMME 上的表现甚至略优于原始模型，这可能归功于融合冗余标记带来的“去噪”效应。
*   **吞吐量优化**：相比基准模型，TTF 实现了约 **1.34倍** 的吞吐量提升，并显著降低了 LLM 的生成延迟。
*   **锚点重要性**：自动选择锚点帧比固定选择首/末帧更稳健，尤其在处理动作密集的短视频（MVBench）时优势明显。

### 7. 优点
*   **即插即用**：无需重新训练模型，可直接集成到现有的 VLM 流水线中。
*   **计算友好**：线性复杂度使其能够扩展到极长视频或高分辨率场景。
*   **兼容性强**：预 LLM 压缩方式使其完美兼容 FlashAttention 等内存高效的注意力内核。
*   **几何一致性**：通过 M-RoPE 重对齐解决了标记压缩后常见的位置感知缺失问题。

### 8. 不足与局限
*   **超参数依赖**：阈值 $t$ 目前仍需手动设置，虽然文中讨论了自适应阈值的潜力，但尚未在当前版本中完全自动化。
*   **极端运动场景**：对于运动幅度极大（超过 3x3 窗口范围）的视频，局部搜索可能会失效，导致关键信息被错误融合。
*   **模型覆盖**：实验主要集中在 Qwen3-VL 系列，尚未在更大规模（如 70B+）或其他架构（如纯 MoE 架构）的 VLM 上进行广泛验证。

## 摘要
视频模型中的 Token 化（通常通过分块实现）会产生过多且冗余的 Token，这严重限制了视频处理的效率和可扩展性。虽然最近基于轨迹的 Tokenizer 通过将视频时长与 Token 数量解耦提供了一种有前景的解决方案，但它们依赖于复杂的外部分割和跟踪流水线，这些流水线运行缓慢且与任务无关。我们提出了 TrajTok，这是一个端到端的视频 Tokenizer 模块，它与视频模型完全集成并针对下游目标进行协同训练，能够根据语义复杂度动态调整其 Token 粒度，且与视频时长无关。TrajTok 包含一个统一的分割器，可在时空维度上对像素进行隐式聚类，从而在单次前向传播中直接生成物体轨迹。通过优先考虑下游适应性而非像素级的分割保真度，TrajTok 保持了轻量和高效，并在实验中提升了视频理解性能。利用 TrajTok，我们实现了一个从头开始训练的视频 CLIP 模型（TrajViT2）。它在分类和检索基准测试中均实现了大规模下的最佳准确率，同时保持了与最先进的 Token 合并方法相当的效率。此外，TrajTok 还被证明是除 Tokenizer 之外的多功能组件。我们展示了它可以无缝集成，既可以作为预训练视觉特征的探测头（TrajAdapter），也可以作为视觉语言模型中的对齐连接器（TrajVLM），在长视频推理方面表现尤为出色。

## Abstract
Tokenization in video models, typically through patchification, generates an excessive and redundant number of tokens. This severely limits video efficiency and scalability. While recent trajectory-based tokenizers offer a promising solution by decoupling video duration from token count, they rely on complex external segmentation and tracking pipelines that are slow and task-agnostic. We propose TrajTok, an end-to-end video tokenizer module that is fully integrated and co-trained with video models for a downstream objective, dynamically adapting its token granularity to semantic complexity, independent of video duration. TrajTok contains a unified segmenter that performs implicit clustering over pixels in both space and time to directly produce object trajectories in a single forward pass. By prioritizing downstream adaptability over pixel-perfect segmentation fidelity, TrajTok is lightweight and efficient, yet empirically improves video understanding performance. With TrajTok, we implement a video CLIP model trained from scratch (TrajViT2). It achieves the best accuracy at scale across both classification and retrieval benchmarks, while maintaining efficiency comparable to the best token-merging methods. TrajTok also proves to be a versatile component beyond its role as a tokenizer. We show that it can be seamlessly integrated as either a probing head for pretrained visual features (TrajAdapter) or an alignment connector in vision-language models (TrajVLM) with especially strong performance in long-video reasoning.

### 1. 核心问题与研究动机
*   **核心问题**：传统的视频模型通常将视频切分为固定的时空分块（Patch），这导致了极高的 Token 冗余度，限制了模型处理长视频的效率和可扩展性。
*   **研究背景**：虽然此前有研究（如 TrajViT）提出基于物体轨迹的 Token 化方案，能有效减少冗余，但它们依赖于外部、不可微且极其缓慢的分割与跟踪流水线（如 SAM2），导致 Token 生成与下游任务脱节，且预处理开销巨大。
*   **研究目标**：开发一种端到端可微、轻量级且能随下游任务自适应调整的轨迹 Tokenizer。

### 2. 核心方法论
TrajTok 由两个核心可微组件组成，旨在单次前向传播中完成轨迹提取：
*   **通用分割器（Universal Segmenter）**：
    *   **隐式聚类**：使用轻量级 Patch 编码器（ConvNeXt）提取特征，并引入一组可学习的查询（Queries）作为聚类原型。
    *   **时空统一处理**：通过 Perceiver 架构，让查询在整个视频序列的特征图上进行交叉注意力计算，利用旋转位置编码（RoPE）处理时空结构。
    *   **软分割掩码**：通过查询与特征图的点积相似度生成软掩码（Soft Mask），确保梯度可回传。
*   **轨迹编码器（Trajectory Encoder）**：
    *   **特征聚合**：根据分割掩码将 Patch 级特征聚合为轨迹 Token。
    *   **自适应 Token 数量**：受 Matryoshka 表示学习启发，每个轨迹可根据复杂度生成不同数量（1、2 或 4 个）的子 Token，以平衡效率与表达力。
*   **损失函数**：结合 Dice Loss 和 Focal Loss 进行监督，侧重于发现所有物体区域而非追求像素级的分类精度。

### 6. 主要结论与发现
*   **性能领先**：TrajViT2 在 K400 上比标准 ViT 提升了 **+4.8%**，在 SSv2 上提升了 **+4.1%**。

### 7. 优点与亮点
*   **端到端可微**：解决了轨迹 Token 化无法与下游任务协同优化的痛点。
*   **通用性强**：既能作为分词器，也能作为适配器或 VLM 连接器，具有极高的工程实用价值。
*   **解耦时长与 Token 数**：实现了 Token 数量由场景语义复杂度决定，而非视频长度决定，极大地优化了长视频处理。

### 8. 不足与局限
*   **分割精度权衡**：为了效率，分割器输出分辨率较低（1/4），可能导致对极小物体的捕捉不够精确。

*   **背景处理**：在简单背景（如 ImageNet）下，由于生成的 Token 过少，性能可能略逊于全量 Patch 模型。

## 摘要
在利用大型多模态模型进行长视频理解的背景下，研究者们已经提出了许多框架。尽管基于 Transformer 的视觉压缩器和记忆增强方法常被用于处理长视频，但它们通常独立地压缩每一帧，因此在需要理解完整事件的任务（如 MLVU 和 VNBench 中的时间排序任务）上表现不佳。这促使我们重新思考从感知到记忆的传统单向方案，转而建立一种反馈驱动的过程，使存储在上下文记忆中的过去视觉上下文能够辅助当前的感知。为此，我们提出了一种用于长视频理解的框架——基于记忆反馈的问题引导视觉压缩（QViC-MF）。其核心是问题引导的多模态选择性注意力（QMSA），它学习从当前视频片段以及记忆中过去的相关帧中保留与给定问题相关的视觉信息。压缩器和记忆反馈对整个视频的每个片段进行迭代工作。这种简单而有效的设计在长视频理解任务上带来了巨大的性能提升。大量实验表明，我们的方法相比当前最先进的方法取得了显著改进，在 MLVU 测试集上提升了 6.1%，在 LVBench 上提升了 8.3%，在 VNBench Long 上提升了 18.3%，在 VideoMME Long 上提升了 3.7%。代码将公开。

## Abstract
In the context of long-term video understanding with large multimodal models, many frameworks have been proposed. Although transformer-based visual compressors and memory-augmented approaches are often used to process long videos, they usually compress each frame independently and therefore fail to achieve strong performance on tasks that require understanding complete events, such as temporal ordering tasks in MLVU and VNBench. This motivates us to rethink the conventional one-way scheme from perception to memory, and instead establish a feedbackdriven process in which past visual contexts stored in the context memory can benefit ongoing perception. To this end, we propose Question-guided Visual Compression with Memory Feedback (QViC-MF), a framework for long-term video understanding. At its core is a Question-guided Multimodal Selective Attention (QMSA), which learns to preserve visual information related to the given question from both the current clip and the past related frames from the memory. The compressor and memory feedback work iteratively for each clip of the entire video. This simple yet effective design yields large performance gains on longterm video understanding tasks. Extensive experiments show that our method achieves significant improvement over current state-of-the-art methods by 6.1% on MLVU test, 8.3% on LVBench, 18.3% on VNBench Long, and 3.7% on VideoMME Long. The code will be released publicly.

### 论文总结：基于记忆反馈的问题引导视觉压缩（QViC-MF）

#### 1. 核心问题与研究动机
*   **核心问题**：现有的长视频理解框架（如基于 Transformer 的压缩器或记忆增强方法）通常采用“感知到记忆”的单向模式，即独立压缩每一帧或每一片段。这种方式忽略了过去视觉背景对当前感知的反馈作用，导致模型在处理需要理解完整事件或复杂时序关系的任务（如时间排序、长程依赖推理）时表现不佳。
*   **研究动机**：为了解决长视频处理中的上下文不一致和时序碎片化问题，作者提出建立一种“反馈驱动”的过程，让存储在记忆中的过去视觉信息能够引导当前的视觉压缩，从而保持事件的完整性。

#### 2. 方法论
QViC-MF 框架采用迭代的片段式处理流程，其核心技术包括：
*   **核心思想**：通过问题引导和记忆反馈，实现“感知-记忆-再感知”的闭环。
*   **关键技术 - QMSA（问题引导的多模态选择性注意力）**：
    *   **掩码（Masking）**：扩展因果掩码，实现帧内压缩的同时允许跨帧的时间上下文交互。
    *   **阻断（Blocking）**：阻断从上下文标记到文本标记的注意力，防止“压缩幻觉”（即文本信息泄露到视觉表示中）。
    *   **引导（Guiding）**：将文本到视觉的注意力权重广播到上下文到视觉的注意力中，使压缩过程具有问题感知能力。
*   **记忆反馈机制**：
    *   **相关性评分**：基于注意力权重计算每帧与问题的相关性得分。
    *   **动态更新**：维护一个固定容量的上下文记忆库，保留得分最高的条目，并剔除低相关性条目。
    *   **反馈循环**：从记忆中检索 Top-Kr 个最相关的历史帧，与当前片段拼接后再次输入压缩器，指导当前感知的精细化。

#### 3. 实验设计
*   **数据集与基准测试**：
    *   **MLVU**：涵盖 3 分钟到 2 小时的多任务长视频理解。
    *   **LVBench**：极长视频理解基准（最高 2 小时）。
    *   **VideoMME**：多领域视频分析，重点评估长视频子集。
    *   **VNBench**：针对“大海捞针”（NIAH）场景的视频任务。
*   **对比方法**：
    *   **闭源模型**：GPT-4o, Gemini 1.5 Pro。
    *   **开源模型**：LLaVA-Video, LongVA, Video-XL, Flash-VStream, MovieChat 等。

#### 4. 资源与算力
*   **算力设备**：使用了 **8 台 NVIDIA H200 GPU** 进行训练。
*   **训练数据**：从 LLaVA-Video-178K 数据集中随机抽取了 5% 的子集（约 83K 样本）进行微调。
*   **模型配置**：基于 LLaVA-Video-7B-Qwen2 架构，使用 LoRA 技术微调视觉压缩器和上下文种子嵌入，解码器 LLM 保持冻结。

#### 5. 实验数量与充分性
*   **实验规模**：作者在四个主流长视频基准上进行了全面测试，并针对 MLVU-test 进行了详细的任务分类对比。
*   **消融实验**：非常充分。涵盖了记忆反馈的有无、QMSA 各组件（M/B/G）的作用、视觉压缩率（1 到 16 个 token/帧）、记忆容量（L=32 到 256）以及反馈帧数量（K vs Kr）的平衡。
*   **客观性**：通过复现 SOTA 模型（如 LLaVA-Video）并使用官方评估脚本，确保了对比的公平性。

#### 6. 主要结论与发现
*   **性能领先**：QViC-MF 在所有测试的开源模型中均达到 SOTA。在 MLVU 测试集上提升 6.1%，在 VNBench Long 上更是取得了 **18.3%** 的巨大涨幅。
*   **效率平衡**：仅使用每帧 16 个视觉 token（远少于 LLaVA-Video 的 169 个），即可实现更高的准确率，证明了压缩算法的有效性。
*   **反馈的重要性**：实验证明，单纯增加记忆而不引入反馈机制对性能提升有限，而“记忆反馈”是提升长时序推理能力的关键。

#### 7. 优点
*   **反馈驱动设计**：打破了传统的单向感知模式，使模型能够根据问题“回看”历史，增强了对稀疏关键事件的捕捉能力。
*   **QMSA 机制**：巧妙地解决了压缩幻觉与问题感知之间的矛盾。
*   **高效率**：在极低的 token 预算下保持了极高的推理精度，适合长视频的扩展。

#### 8. 不足与局限
*   **计数任务瓶颈**：在 MLVU 的 Action Count 和 VNBench 的计数任务中表现提升不明显。作者分析认为这可能是解码器 LLM 本身的推理限制，而非视觉检索的问题。
*   **任务依赖性**：对于需要全局总结（如摘要）的任务，反馈机制带来的增益小于针对特定稀疏事件检索的任务。
*   **实时性挑战**：虽然复杂度是线性的，但迭代反馈过程在处理极长视频时仍存在一定的计算开销。

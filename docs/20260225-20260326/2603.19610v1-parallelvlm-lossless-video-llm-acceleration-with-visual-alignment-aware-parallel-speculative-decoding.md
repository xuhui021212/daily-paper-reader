## 摘要
尽管目前的视频大语言模型（Video-LLMs）在视频理解任务中取得了令人印象深刻的性能，但其自回归解码效率仍受限于海量的视频 token 数量。视觉 token 剪枝可以部分缓解这一瓶颈，但现有方法仍面临信息丢失的问题，且在解码加速方面效果有限。在本文中，我们提出了 ParallelVLM，这是一个无需训练的“先草拟后验证”投机解码框架，它克服了长视频场景下草拟模型（draft model）与目标模型（target model）之间相互等待以及加速比受限的问题。ParallelVLM 具有两个并行化阶段，可最大化硬件利用率，并引入了一种无偏验证器引导剪枝（Unbiased Verifier-Guided Pruning）策略，通过消除注意力引导剪枝中的位置偏差，更好地对齐草拟模型和目标模型。大量实验表明，ParallelVLM 在保持高接受长度的同时，有效地将草拟窗口扩大了 1.6~1.8 倍；与原生自回归解码相比，在 LLaVA-Onevision-72B 上将各种视频理解基准测试加速了 3.36 倍，在 Qwen2.5-VL-32B 上加速了 2.42 倍。

## Abstract
Although current Video-LLMs achieve impressive performance in video understanding tasks, their autoregressive decoding efficiency remains constrained by the massive number of video tokens. Visual token pruning can partially ease this bottleneck, yet existing approaches still suffer from information loss and yield only modest acceleration in decoding. In this paper, we propose ParallelVLM, a training-free draft-then-verify speculative decoding framework that overcomes both mutual waiting and limited speedup-ratio problems between draft and target models in long-video settings. ParallelVLM features two parallelized stages that maximize hardware utilization and incorporate an Unbiased Verifier-Guided Pruning strategy to better align the draft and target models by eliminating the positional bias in attention-guided pruning. Extensive experiments demonstrate that ParallelVLM effectively expands the draft window by $1.6\sim1.8\times$ with high accepted lengths, and accelerates various video understanding benchmarks by 3.36$\times$ on LLaVA-Onevision-72B and 2.42$\times$ on Qwen2.5-VL-32B compared with vanilla autoregressive decoding.

*   **视觉 Token 剪枝：** 虽然能提速，但属于有损压缩，会导致细粒度理解能力的下降。
*   **投机解码（Speculative Decoding, SD）：** 虽然无损，但在视频场景下存在“串行执行瓶颈”（草稿模型和目标模型互相等待）以及“剪枝对齐难题”（为了让草稿模型更快而进行的剪枝往往破坏了其与目标模型的输出一致性）。

**研究动机：** 开发一种既能实现大幅度加速，又能保证输出完全无损，且能有效处理海量视频 Token 的并行推理框架。

#### 2. 论文提出的方法论
论文提出了 **ParallelVLM**，其核心思想是通过并行化流水线和无偏见的剪枝策略来提升效率。

*   **核心技术细节：**
    1.  **并行预填充（Parallel Prefilling, PP）：** 目标模型进行全量 Token 预填充的同时，草稿模型同步进行剪枝后的 Token 预填充。通过利用目标模型前几层的中间表示来指导草稿模型的剪枝，消除了草稿模型的启动开销。
    2.  **并行解码（Parallel Decoding, PD）：** 采用重叠窗口机制。当目标模型在验证第 $i-1$ 组草稿 Token 时，草稿模型同步生成第 $i$ 组 Token，从而最大化硬件利用率。
    3.  **无偏验证器引导剪枝（UV-Prune）：** 针对传统注意力引导剪枝存在“位置偏差”（倾向于保留序列两端 Token）的问题，UV-Prune 计算视觉 Token 与文本查询在目标模型浅层间的**余弦相似度变化量**。变化量大的 Token 被认为携带了关键语义，以此指导草稿模型的 Token 筛选。

*   **解决位置偏差：** UV-Prune 是一种创新的剪枝准则，解决了视频 Token 处理中常见的“注意力汇聚”导致的误剪枝问题。

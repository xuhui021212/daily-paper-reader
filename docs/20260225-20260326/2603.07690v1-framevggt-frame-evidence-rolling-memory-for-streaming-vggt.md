## 摘要
流式视觉几何 Transformer（如 StreamVGGT）能够实现强大的在线 3D 感知，但面临 KV 缓存（KV-cache）无限增长的问题，这限制了其在长流数据上的部署。我们从几何支撑的角度重新审视了有限内存下的流式处理。在几何驱动的推理中，记忆质量不仅取决于保留了多少 token，还取决于保留的记忆是否仍能维持足够连贯的局部支撑。这表明在固定预算下，token 级别的保留策略可能不再适用，因为它会稀释每个贡献帧中可用的证据，并使后续融合对弱对齐的历史信息更加敏感。基于这一观察，我们提出了 FrameVGGT，这是一个帧驱动的滚动显式记忆框架，它将每一帧的增量 KV 贡献视为一个连贯的证据块。FrameVGGT 将每个块总结为紧凑的原型，并在严格的预算下维护一个固定容量的互补帧块中期库，同时为罕见的长时间性能退化提供可选的轻量级锚点层。在长序列 3D 重建、视频深度估计和相机位姿基准测试中，FrameVGGT 在有限内存下实现了良好的准确度与内存权衡，同时在长流数据中保持了更稳定的几何性能。

## Abstract
Streaming Visual Geometry Transformers such as StreamVGGT enable strong online 3D perception but suffer from unbounded KV-cache growth, which limits deployment over long streams. We revisit bounded-memory streaming from the perspective of geometric support. In geometry-driven reasoning, memory quality depends not only on how many tokens are retained, but also on whether the retained memory still preserves sufficiently coherent local support. This suggests that token-level retention may become less suitable under fixed budgets, as it can thin the evidence available within each contributing frame and make subsequent fusion more sensitive to weakly aligned history. Motivated by this observation, we propose FrameVGGT, a frame-driven rolling explicit-memory framework that treats each frame's incremental KV contribution as a coherent evidence block. FrameVGGT summarizes each block into a compact prototype and maintains a fixed-capacity mid-term bank of complementary frame blocks under strict budgets, with an optional lightweight anchor tier for rare prolonged degradation. Across long-sequence 3D reconstruction, video depth estimation, and camera pose benchmarks, FrameVGGT achieves favorable accuracy--memory trade-offs under bounded memory, while maintaining more stable geometry over long streams.

### 1. 核心问题与整体含义（研究动机）
*   **研究背景**：流式视觉几何 Transformer（如 StreamVGGT）在在线 3D 重建、深度估计和位姿跟踪方面表现出色，但其 **KV 缓存（KV-cache）随序列长度无限增长**，导致长视频处理时内存和延迟不可控。
*   **核心痛点**：现有的解决方案（如 InfiniteVGGT）采用 **Token 级别的保留策略**。作者发现，几何推理（如三角测量、位姿估计）依赖于局部连贯的观测证据，而非孤立的 Token。Token 级剪枝会稀释单帧内的几何支撑，导致证据碎片化，使模型对噪声和弱对齐历史更加敏感。
*   **研究动机**：提出一种“支撑对齐”（Support-aligned）的内存管理策略，将保留单位从 Token 提升到“帧/块”级别，以保持几何推理所需的连贯性。

### 2. 方法论：核心思想与关键技术
FrameVGGT 将 Transformer 的内部 KV 缓存重新组织为以帧为驱动的滚动显式记忆。
*   **核心思想**：将每一帧产生的增量 KV 贡献视为一个不可分割的“证据块”，通过维护一个固定容量的“中期库”和“全局锚点层”来管理内存。
*   **关键技术细节**：
    *   **块原型总结（Prototype Summarization）**：为了高效比较，将每帧的 KV 块总结为一个轻量级的键空间原型（Key-space Prototype），即对该帧所有 Token 的 Key 取均值并进行 $\ell_2$ 归一化。
    *   **中期库（Middle Bank）管理**：使用基于余弦相似度的**度量 k-中心（Metric k-center）目标函数**。在实践中采用增量贪婪最远优先（Greedy Farthest-first）策略，保留最具代表性和互补性的帧块，避免冗余（如相机静止时的重复观测）。
    *   **全局锚点层（Anchor Tier）**：针对快速旋转或严重遮挡等极端情况，保留极少数（如 4 帧）具有高几何可靠性（置信度+清晰度）和空间新颖性的长期参考帧。
    *   **多尺度组织**：内存分为近期、中期和长期锚点，形成不同时间尺度的证据支撑。

*   **任务与数据集**：

*   **实验规模**：论文涵盖了三大核心 3D 视觉任务，并在多个标准数据集上进行了长序列（最高 1000 帧）测试。

### 6. 主要结论与发现
*   **粒度匹配至关重要**：在固定内存预算下，保持帧级证据的完整性比追求 Token 级的全局覆盖对几何推理更有利。
*   **性能与效率双赢**：FrameVGGT 在显著降低内存占用的同时，减少了长序列中的漂移和伪影，实现了更好的准确度-内存权衡。
*   **近期冗余风险**：过度分配内存给最近的帧（Recent-K）会导致有效基线多样性下降，削弱长程约束。

### 7. 优点（亮点）
*   **理论深刻**：从几何支撑（Geometric Support）的角度指出了 Token 级剪枝的结构性缺陷（支撑稀释、时空解耦）。
*   **即插即用**：无需训练，直接提升现有流式 Transformer 的长序列处理能力。
*   **鲁棒性强**：通过双层内存设计（中期库+锚点），兼顾了局部连续性和全局一致性。

### 8. 不足与局限
*   **策略固定**：目前的内存分配比例（如中期库与锚点的数量）是手动指定的，尚未实现根据场景复杂度或运动状态动态调整的自适应策略。
*   **计算开销**：虽然内存大幅降低，但每帧仍需计算原型并进行贪婪选择，虽然开销较小，但在极端实时性要求的场景下仍需进一步优化。
*   **应用限制**：主要针对几何类任务（深度、位姿），对于纯语义类的流式任务，Token 级剪枝是否仍劣于帧级保留尚待验证。
